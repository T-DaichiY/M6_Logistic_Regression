🔧 RでAICを使う：2つのKに注意

RのAIC()関数の引数kと数式のKの違い。デフォルトk=2が古典的AIC。NYCフライトデータで動作を確認。

① 数式のKとRのkの違い② AIC()関数の動作③ 実践上の注意点📝 Quiz
①

AICの公式をおさらい

数式のKとRのkは別物

AICの公式：

AIC = −2 × log-likelihood + 2K
  • ここで K（大文字） = モデルのパラメータ数（予測変数 + 切片）
  • 2 = パラメータ1つあたりのペナルティの大きさ（古典的AIC）

Rの AIC() 関数を使うとき、引数 k（小文字）が出てきます。これが混乱の原因です。

⚠️ 重要：数式のK（パラメータ数）とRの引数k（ペナルティの大きさ）は別物！同じ文字だが意味が違います。
②

RのAIC()関数の動作

引数kはペナルティ（デフォルト2）

Rの AIC(model, k=...) の k は「1パラメータあたりのペナルティ」です（デフォルト = 2）。

# NYC Flights データで回帰モデルを作成library(nycflights13)delay_reg <- lm(arr_delay ~ air_time + distance, data = flights)# 通常のAIC（kを指定しない → デフォルトk=2）AIC(delay_reg)# → 2,720,944# k=2 を明示的に指定（同じ結果）AIC(delay_reg, k = 2)# → 2,720,944（変わらない）# k=3 に変更（ペナルティを強化）AIC(delay_reg, k = 3)# → 2,720,949（少し大きくなる）
AIC(k=2) = 2,720,944 AIC(k=3) = 2,720,949 ← kを3にするとペナルティが強まる
③

実践上の注意点

課題ではkを変えないこと
📌 KとkとBICの関係：数式の K は「パラメータの個数」、Rの k は「1個あたりの罰金の重み」で別物です。k=2 が古典的AIC。もし k=log(n)（nは標本数）にすると、それはBIC（ベイズ情報量規準）に相当し、大標本ではAICより厳しくパラメータを罰します。課題ではデフォルトのk=2で統一しましょう。

課題で指示がない限り、k は変更不要です。デフォルト（k=2）が「古典的AIC」です。

状況対応
通常の分析AIC(model)　← k=2がデフォルト
課題で「古典的AICを使え」と指示AIC(model)　← 何も変えなくてよい
BICを使いたい場合AIC(model, k=log(n))　← kにlog(n)を指定
📌 Rがパラメータ数を自動でカウント：AIC()にモデルオブジェクトを渡すだけで、Rが予測変数の数を自動的に数えてくれます。手動でKを指定する必要はありません。
🍎 かんたん説明

RのAIC()は「k=ペナルティの強さ」で、数式のK（パラメータ数）ではありません。デフォルトk=2がほとんどのケースで使う古典的AICなので、基本的に何もしなくてOKです🙆

📖 用語集

AIC()関数のk
k in AIC()
ペナルティの強さ（デフォルト2）。数式のK（パラメータ数）とは別物。
古典的AIC
Classical AIC
ペナルティが2K（k=2）の標準的なAIC。Rのデフォルト。
BIC
Bayesian Information Criterion
AIC(model, k=log(n))で計算。サンプル数に応じてペナルティが変わる。
パラメータ数
Number of Parameters
切片 + 予測変数の数。Rが自動的にカウントする。
