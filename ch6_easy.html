🍎 第6章 超やさしい版：モデル評価②：AIC

AIC = −2LL + 2K。ペナルティが逸脱度との違い。変数を増やすと何が起きるかをスライダーとRコードで体感しよう。

一言でいうとペナルティを体感Rでモデル比較📊 本編はこちら
⚡

一言でいうと

AIC = 当てはまりの良さ − 複雑さへのペナルティ

🎯 これだけ覚えればOK

AIC = −2LL + 2K。当てはまりが良い（逸脱度が小さい）のにパラメータが少ない（Kが小さい）ほどAICは低くなる。AICは小さいほど良いモデル。逸脱度だけではモデルが複雑になるほど必ず良くなってしまうため、Kでペナルティを課す。

重回帰のR²に相当する「モデルの良さの指標」がAICです。ただし方向が逆で、小さいほど良いという点に注意しましょう。

①

ペナルティを体感する

変数を増やすとAICはどう変わるか

スライダーでKを増やすとペナルティ（2K）が増え、AICが上がります。意味のない変数を追加すると逸脱度の改善よりペナルティが大きくなり、AICが悪化します。

②

Rでモデル比較する

AIC()関数の使い方

▶ AICを手計算で確認（仕組みを理解する）

# AICの計算式（手計算確認）
log_lik <- logLik(model)  # 対数尤度
k <- length(coef(model))
aic_manual <- -2 * as.numeric(log_lik) + 2 * k
# R組み込みのAIC()と一致するか確認
AIC(model)  # ← 同じ値になる
aic_manual      # ← 手計算でも同じ

▶ 2モデルをAICで比較する

# 2つのモデルを比較する
model1 <- glm(Sex ~ Diameter,             data=Abalone, family=binomial)
model2 <- glm(Sex ~ Diameter + Rings,     data=Abalone, family=binomial)
AIC(model1)  # → 3597.8
AIC(model2)  # → 3596.1（わずかに低い → model2が少し良い）
# AICが低い方を選ぶ

🤔 答えられるかな？（クリックで答えが開く）

Q1. AICの計算式は？A. AIC = −2LL + 2K。−2LLは逸脱度、Kはパラメータ数（切片＋予測変数の数）。
Q2. AICはどちらが良い？A. 小さいほど良い。最もAICが低いモデルを選ぶ。差が2未満ならほぼ同じと判断。
Q3. 逸脱度だけで比べてはダメな理由は？A. 変数を増やすほど逸脱度は必ず下がる（過学習）。AICはペナルティ（2K）で無意味な変数の追加を防ぐ。

💡 まとめ

• AIC = −2LL + 2K（小さいほど良い）

• Kはパラメータ数（切片+変数）

• 変数増やす=ペナルティ増える

• AIC差が2未満ならほぼ同じ
