Recap · これまでの話

まずは単回帰のおさらい

このモジュールでこれまで扱ってきたのは単回帰（単純線形回帰／最小二乗回帰）。 手がかりがたった1つ——たとえば「気温」だけからレンタル数を予測する、一番シンプルな直線の式でした。

Yᵢ = B0 + B1·Xᵢ + 誤差 E

ある観測 i について、予測したい Yᵢ（レンタル数）を、手がかり Xᵢ（気温）から、 切片 B0 と傾き B1 で表します。推定には必ずズレ（ばらつき）が残るので、最後に誤差 E がつきます。

📌 講義の小ネタ：E の代わりに Ŷ
誤差 E を式に書く代わりに、左辺を Ŷ（ワイハット＝予測値）にして書くこともできます。 「実測 Y」と「予測 Ŷ」を区別するのが、このあとの残差の話につながります。
🍎 かんたん説明
単回帰は「アイスの売上は気温だけで決まる！」と決めつけて予測するようなもの。 でも本当は雨の日や曜日も効きますよね。その“他の手がかり”も全部使おう、というのが次の重回帰です。
The idea · 重回帰とは

「気温だけじゃなく、
もっと使おうよ」

重回帰の主張はシンプルです。講師の言葉そのまま—— 「気温からレンタル数を予測するんじゃなくて、もっとたくさんの予測変数を使おうよ」。 式は単回帰そっくり。最初の部分は同じで、そこに X が増えていくだけです。

Y = B0 + B1X1 + B2X2 + B3X3 + … + BnXn + 誤差

最初の B0+B1X1 は単回帰そのまま。そこに X2, X3… と 第n番目の予測変数まで続けられます（添え字は予測変数 Predictor の頭文字を取って P で書くこともあります）。 手がかりを増やすほど、応答変数 Y（＝レンタル数）の推定がよりよくなる——というのが狙いです。 講師いわく「湿度と気温と風速がわかれば、より良い推定ができる」。

探偵が事件を解くとき、手がかりは1つより多いほうが正しい犯人に近づけます。 重回帰は「湿度」「気温」「風速」という複数の手がかりを集めて、より正確に「今日のレンタル数」を当てる探偵です🕵️
In R · モデルを作る

Rでは + でつなぐだけ

重回帰モデルを作るのは、講師いわく「驚くほど簡単」。 応答変数 ~ 予測変数1 + 予測変数2 + … と書くだけです。~（チルダ）は「〜で説明する／〜が与えられたとき」の意味。 data = bikes は式の最初でも最後でも構いません。

1# rentals（レンタル数）を 湿度+気温+風速 で予測する
2bikes_model_1 <- lm(rentals ~ humidity + temperature + windspeed,
3 data = bikes)
4bikes_model_1 # 係数（coefficients）が出る
5summary(bikes_model_1) # 係数＋詳しい情報

実行すると係数（coefficients）が出ます。予測変数が3つなので B₁, B₂, B₃。これがあなたのモデルの式になります。 summary() の「Estimate（推定値）」列に並ぶのが、回帰式に使う係数。 それ以外の部分は単回帰の summary とほぼ同じで、予測変数の行が増えただけです。

📊 4変数を1枚の図で：plotly の3D散布図
予測変数3つ＋応答変数1つ＝合計4変数は、plotly ライブラリ（要インストール）で1つの3D散布図にできます。 X・Y・Z軸に湿度・気温・風速、点の色（marker）をレンタル数に。ドラッグして回転でき、講義では 低い値（左下・暗い色）＝レンタル少、黄色＝レンタル多。気温が低いと色が変わる関係が立体的に見えます。
Coefficients · 係数の読み方

3つの数字を教えてくれれば、
予測できる

得られた係数を使うと、回帰式はこう書けます。

予測レンタル数 (rental_hat) = 切片 (B₀)
  + B₁ × 湿度(humidity) + B₂ × 気温(temperature) + B₃ × 風速(windspeed)

つまり、ある日の湿度・気温・風速の3つさえ教えてもらえれば、その日のレンタル数を予測できる、ということです。

⚠️ 係数の大きさ＝重要度ではない
講義のモデルでは 湿度の係数が −3100 で、3つの中で一番大きな推定値です。 では「湿度が一番重要」？ ── そうとは言えません。湿度・気温・風速は単位（スケール）が違ううえ、 しかも3変数とも p値はとても小さい（みんな有意）。「どれが一番効くか」を公平に比べる方法は 第3章（標準化回帰係数）で学びます。
🍎 かんたん説明
回帰式は「料理のレシピ」。「湿度を◯倍、気温を◯倍、風速を◯倍して全部足す」というレシピ通りに計算すれば、 その日のレンタル数という料理ができあがります🍳

残差（ざんさ）＝予測と実際のズレ

手計算の予測は約3,300件、実際は1,248件。差はおよそ2,000件。予測が実際より大きい（過大評価）ので、これは大きな負の残差です。 講師いわく「この日についてはモデルがあまり役立っていない」。

予測 Ŷ（過大評価）3,300
実際 Y1,248
残差 = 実際 − 予測 ≈ 1,248 − 3,300 = −2,052（大きな負の残差）
🍎 かんたん説明
3つの予測方法は「電卓のレベルアップ」。手計算は紙と鉛筆、 fitted.valuesは「過去の結果一覧表」から探すだけ、 predictは「どんな数字を入れても答えを出す万能電卓」。だから普段は predict でOK！

Glossary · 重要用語

この章の用語

重回帰分析
Multiple Regression
2つ以上の予測変数で1つの量的な応答変数を予測する手法。
予測変数
Predictor / Independent
予測の手がかり（X）。説明変数・独立変数とも。添え字はPで書くことも。
応答変数
Response / Dependent
予測したい対象（Y）。例：レンタル数。
係数
Coefficient (B₀,B₁…)
各予測変数にかける数字。summaryのEstimate列に出る。
残差
Residual
実際−予測。マイナスなら過大評価、プラスなら過小評価。
fitted.values
あてはめ値
データに実在する行の予測値。任意の値には使えない。
predict関数
predict()
データに無い値でも予測できる最も便利な方法。newdataはdata.frameで。
