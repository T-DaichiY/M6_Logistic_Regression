①

ロジスティック回帰とは？

重回帰との決定的な違い：予測する結果が違う

M5では量的な値（自転車レンタル数）を予測する重回帰を学びました。ロジスティック回帰は似ていますが、予測したいものが根本的に異なります。

結果が「どちらか一方（バイナリ）」——0か1か——のときに使います。

手法予測するもの例
重回帰量的な連続値レンタル数・売上・気温
ロジスティック回帰確率（0〜1）→ 2択の判定がんか否か・スパムか否か・合格か否か
🍎 かんたん説明

重回帰は「何個売れるか？」を予測。ロジスティック回帰は「売れるか？売れないか？」を予測します。料理で言えば、重回帰は「何グラム？」、ロジスティック回帰は「おいしいか？まずいか？」🍳

②

バイナリ結果と確率の考え方

0と1のラベルが何を意味するか

バイナリ（binary）とは「2択」のことです。ロジスティック回帰では、結果変数を次のようにラベルします：

  • 0（ゼロ）：「失敗」「なし」「いいえ」のグループ
  • 1（いち）：「成功」「あり」「はい」のグループ

ロジスティック回帰が予測するのは「1である確率」です。例えば：

🔬
がんの診断
1=悪性　0=良性
📧
スパム検出
1=スパム　0=正常
🎓
合否予測
1=合格　0=不合格
💳
信用審査
1=良い　0=悪い
📌 注意：「バイナリロジスティック回帰」が基本です。3つ以上のカテゴリがある場合は「多項ロジスティック回帰（multinomial）」になりますが、このコースでは扱いません。
③

アバロンデータで実践

オスかメスかを予測する

このモジュールではアバロン（鮑）のデータセットを使います。AppliedPredictiveModelingパッケージに含まれています。

library(AppliedPredictiveModeling)data(abalone)# 変数の確認：Type（オス/メス/不明）+ 各種計測値head(abalone)# オスとメスだけに絞る（不明 = I を除外）abalone2 <- abalone[abalone$Type != "I", ]

次に、ロジスティック回帰を実行します。重回帰の lm() の代わりに glm()（Generalized Linear Model）を使います。

# family = binomial が「ロジスティック回帰」の指定model1 <- glm(Type ~ Diameter, data = abalone2, family = binomial)summary(model1)
⚠️ lm() ではなく glm() を使う！　family = binomial を忘れずに指定してください。これがロジスティック回帰であることをRに伝える重要な設定です。
🍎 かんたん説明

glm()は「ゼネラル（汎用的）な線形モデル」の略。lm()が「直線専用」とすると、glm()は直線以外の曲線も含む万能な兄貴分です💪

④

なぜ重回帰をそのまま使えないのか

確率は0〜1の間でないといけない

「0か1か」の結果変数に対してそのまま重回帰を使うと、予測値が0より小さくなったり1より大きくなったりします。これは確率としておかしい。

重回帰の予測値
−∞ 〜 +∞
❌
確率としての要件
0 〜 1

ロジスティック回帰はシグモイド関数という特殊な変換を使って、どんな入力値でも必ず0〜1の間に収まる確率を出力します。

回帰式の値
−∞ 〜 +∞
→
シグモイド関数
1 / (1 + e⁻ˣ)
→
確率
0 〜 1
🍎 かんたん説明

シグモイド関数は「S字カーブ」の形をしています。どんな大きい数も小さい数も、必ず0と1の間に「押し込んで」くれる魔法の変換です🔄

📖 用語集

バイナリ結果
Binary Outcome
0か1かの2択の結果変数。ロジスティック回帰が予測する対象。
glm()
Generalized Linear Model
Rで一般化線形モデルを実行する関数。ロジスティック回帰にはfamily=binomialを指定。
シグモイド関数
Sigmoid Function
S字カーブを描く関数。どんな値も0〜1の確率に変換する。
family=binomial
Binomial Family
glm()でロジスティック回帰を指定するための引数。
