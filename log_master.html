基礎

対数は「何回かけたか?」を聞いている

指数の逆向きの質問です。「2を3回かけたら8」が指数なら、「8は2を何回かけた数?→3」が対数。

log2 8 = 3  ⇔  23 = 8

かけ算の世界を、たし算の世界へ

かけ算の世界対数をとると意味
log(A × B)= logA + logBかけ算 → たし算
log(A ÷ B)= logA − logB割り算 → 引き算
log(Aⁿ)= n × logA累乗 → かけ算
底の3種 log₁₀(桁を数える) / log₂(情報・コンピュータ) / ln=logₑ(自然対数, e≈2.718)。
⚠️真数は必ず正。log(0)・log(負)は存在しない。「何回かけても0や負にはならない」から。
対数グラフ

軸を変えると、潰れた関係が直線になる

指数的に増えるデータは線形軸だと序盤が潰れます。y軸を対数にすると一直線に。

なぜ直線? y=a·bˣ の両辺 log で log y = log a + x·log b。log y を縦軸にすれば x の一次関数=直線。

線形・片対数・両対数の見分け

軸直線になる式正体
線形×線形y=ax+b一次関数
片対数y=a·bˣ指数(感染・複利)
両対数y=a·xⁿべき乗則(スケール則)
⚠️対数軸に0は描けない・中央は√10≈3.16・急増が緩やかに見える錯覚に注意。
R実データ・ハンズオン

グラフの傾きから、式を復元する

対数グラフの真価は「見て終わり」ではありません。直線の傾きと切片を読めば、元の式の係数が逆算できる。これが分析の核心です。

CASE A · 片対数指数データ y = a·bˣ の復元

下のデータ(x=0..10)を片対数で描くと直線に。傾き = log₁₀(b)、切片 = log₁₀(a) から元の a, b が出ます。スライダーで直線化を体感し、ボタンで復元してみてください。

# R：傾きから指数式を復元
x <- 0:10
y <- c(3, 5.1, 8.7, 14.7, 25.1, 42.6, 72.4, 123.1, 209.3, 355.8, 604.8)
fit <- lm(log10(y) ~ x) # log10(y) を直線回帰
b <- 10^coef(fit)[2] # 傾き → b
a <- 10^coef(fit)[1] # 切片 → a
# 結果: a ≈ 3.00, b ≈ 1.70 → y = 3.00 × 1.70^x

CASE B · 両対数べき乗則 y = a·xⁿ の復元

体重と代謝量のようなべき乗則は両対数で直線に。このとき 傾きそのものが指数 n、切片が log₁₀(a)。傾きが「スケール則の指数」を直接教えてくれます。

# R：両対数の傾き = べき指数 n
x <- 1:10
y <- c(2, 5.66, 10.39, 16, 22.36, 29.39, 37.04, 45.25, 54, 63.25)
fit <- lm(log10(y) ~ log10(x)) # x も log に
n <- coef(fit)[2] # 傾き = n
a <- 10^coef(fit)[1]
# 結果: n ≈ 1.50, a ≈ 2.00 → y = 2.00 × x^1.5
この手順が"使える"理由 実データを片対数・両対数で描き、どちらで直線になるかを見れば、指数モデルかべき乗モデルかが判定できる。 さらに傾きから係数まで取り出せる。これは lm() の前にやる、強力なEDAの型です。
⚠️log10(y) は 0・負があると NA / 警告。log1p() 等で前処理を。直線回帰の前に必ず分布を確認。
感染症モデル

指数 → ロジスティック。対数が"曲がる瞬間"を暴く

感染初期は N₀·e^{rt} の指数増殖。しかし感受性人口が尽きると頭打ち(ロジスティック)。線形では潰れて見えませんが、対数軸では「直線がいつ折れたか」=ピークアウトの瞬間がはっきり読めます。

# ロジスティック：N(t) = K / (1 + ((K−N₀)/N₀)·e^{−rt})
K <- 5000; N0 <- 5; r <- 0.9
t <- 0:20
N <- K / (1 + ((K-N0)/N0) * exp(-r*t))
ggplot(data.frame(t,N), aes(t,N)) + geom_line() + scale_y_log10()
局面対数グラフの見え方意味
初期直線(右肩上がり)指数増殖・増加率 r 一定
中盤傾きが寝てくる感受性人口が減り減速
後期水平に漸近収束(Kに到達)
なぜコロナ報道は対数軸だったか 指数局面では「直線かどうか」で増加率の一定性が判断でき、傾きが折れた瞬間=対策が効き始めた合図として読めるから。線形軸では序盤の変化が潰れて、この折れ目が見えません。
⚠️初期の指数フィットで未来を外挿すると過大予測になる。赤い破線(指数予測)が実際を大きく上回るのがその証拠。モデルの適用範囲に注意。
O(log n) · 9618教材

二分探索：100万件を、約20回で射抜く

毎回「半分」を捨てるから、データ量 n が増えてもステップ数は log₂ n でしか増えません。これが O(log n) の正体。下で実際に半分ずつ消えていく様子を動かせます。

ターゲット を探します。「次のステップ」を押してください

n が増えても、ステップは log でしか伸びない

線形探索は n に比例して伸びるが、二分探索はほぼ寝たまま。これが対数の威力
データ件数 n線形探索 O(n)二分探索 ⌈log₂(n+1)⌉
10最大 10 回4 回
100最大 100 回7 回
1,000最大 1,000 回10 回
1,000,000最大 100万 回20 回
1,000,000,000最大 10億 回30 回
9618 で問われるポイント 二分探索は整列済み配列が前提。各ステップで探索範囲が半分になるため、最悪計算量は O(log n)。 「n が1000倍になっても、ステップは約10回増えるだけ」と説明できると、対数の意味が腑に落ちます。
⚠️未整列データには使えない。整列コストが別途かかる点も、計算量を語るときの典型的な引っかけ。
