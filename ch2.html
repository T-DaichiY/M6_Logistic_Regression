Chapter 2 · Diagnostics

モデルは作れた。
でも良いモデルか、
どう見抜く？

重回帰では予測変数の候補がたくさん。4つ入りより2つだけのほうが良いこともあります。 講師の鉄則は「1つの指標（特にp値だけ！）に頼らず、ホリスティック（全体的）に判断する」。 気温だけのモデルと、気温＋湿度＋風速のモデルを、6つの指標で“健康診断”します。

Why · なぜ診断が必要か

1つの指標に頼らない

重回帰には予測変数の候補がたくさん。でも「4つ入り」より「2つだけ」のほうが良いこともあります。 では、どのモデルを選ぶべきか、本当に良い予測ができているのか？それを見抜くのが「診断」です。

⚠️ p値だけに頼らない（統計のよくある落とし穴）
講師いわく、統計では summary の末尾にある星（★ ★★ ★★★＝小さなp値）だけを見て「有意だからOK」と決めてしまいがち。 でも診断は複数の指標を総合して（ホリスティックに）判断するのが鉄則です。

診断には多くの道具があります。残差分析（quick plot で出る残差プロット）、影響点分析（第3.1章）、 VIF／多重共線性（第4章）は別の章で扱います。この章で扱うのは、モデルAとBのどちらが良いかを比べる“中間の指標たち”です。

モデルA（単回帰）

  • 気温だけ
  • シンプル
  • 説明力は？

モデルB（重回帰）

  • 気温＋湿度＋風速
  • 情報は多い
  • 本当に良くなった？

この2つを、これから6つの指標で“健康診断”します。

🍎 かんたん説明
診断はお医者さんの「健康診断」。体温だけ見て健康とは言えません。身長・体重・血圧・心拍… いろんな数値を合わせて見るから本当の状態がわかります。モデルも同じです🩺
調整済みR²（高いほど良い ⬆️）

無駄な変数を足すと、
2つのR²は逆方向に動く

ふつうのR²の弱点は、役に立たない変数を足してもR²が上がってしまうこと。 講師の例：家の価格を予測するのに「ドアノブの数」や「ドアノブの平均直径」を入れても、価格とは無関係なのにR²は上がる。 調整済みR²は、こうした役立たない予測変数にペナルティを与えて、水増しされたR²を下げます。

1# 調整済みR²を取り出す（高いほうが良い）。$ で直接アクセス
2summary(temp_model)$adj.r.squared # 気温だけ
3summary(bikes_model_1)$adj.r.squared # 気温+湿度+風速（少し増えた）

結果、重回帰のほうが調整済みR²が（大きくはないが）増えました。講師いわく「湿度と風速を知ることで、レンタル数の予測がよくなる」。 ただし、ここで無駄な変数を足すと、上のグラフのように調整済みR²は逆に下がっていきます。

🍎 かんたん説明
R²は「荷物をいくら詰めても文句を言わないカバン」。調整済みR²は「いらない荷物を入れると重くなって怒るカバン」。 だから本当に必要なものだけ入れるようになります🎒
AIC & BIC（低いほど良い ⬇️）

「当てはまり」と「シンプルさ」の
バランスを採点する

AIC（赤池情報量規準）は、データへの当てはまりの良さとモデルの複雑さのバランスを取ろうとする指標。 データによく合うけれど、複雑すぎて過学習にならないモデルを好みます。

AIC

  • 当てはまり × 複雑さ のバランス
  • 低いほど良い ⬇️
  • 相対指標：絶対値（例10,000）に意味はなく、他モデルと比べるためだけに使う
  • 式はノートブック末尾の「formulas」に。暗記不要（興味があれば導出を参照）

BIC

  • AICの仲間。複雑さへの罰がもっと強い
  • よりシンプルな（パーシモニアスな）モデルを好む
  • Rでは大文字 BIC()（小文字はエラー）
  • 別名 SBC とも呼ばれる
1AIC(temp_model) # 気温だけ
2AIC(bikes_model_1) # 気温+湿度+風速 ← こちらが少し低い＝良い
3BIC(bikes_model_1) # BICでも重回帰が好まれる（lowercase はエラー）
📌 AIC vs BIC
どちらも「低いほど良い」。違いはBICのほうがシンプルさを強く求めること。変数が多いモデルでは、BICはAICより少ない変数を選びがちです。
🍎 かんたん説明
AICとBICは「テストの採点者」。当てはまりが良いと加点、答案が長すぎ（複雑すぎ）ると減点。 BICは特に厳しい先生。点数は低いほど良い＝ゴルフのスコアと同じ⛳
S & RMSE（低いほど良い ⬇️）

予測は、実際の点から
どれだけ離れている？

残差は「回帰線から観測点までの距離」。距離が大きいほど誤差が大きい。 だから残差標準誤差 S は小さいほど良い。すべての点が線の近くにある＝良いモデルです。 summary では Residual standard error: 1509 のように出ます。

1# summary の "Residual standard error"（例: 1509）。$sigma で取り出す
2summary(bikes_model_1)$sigma # 重回帰のほうが小さい＝良い

RMSE（二乗平均平方根誤差）は自作する

Rに RMSE の組み込み関数は無いので関数を作ります。やることは名前の通り： 残差（実際−予測）を二乗 → 平均 → 平方根（Root-Mean-Square-Error）。 最後に平方根を取るのは、二乗で変わった単位を元に戻すためです。

1calculate_rmse <- function(actual, predicted) {
2 residuals <- actual - predicted # 残差
3 sqrt(mean(residuals^2)) # 二乗→平均→平方根
4}
5calculate_rmse(bikes$rentals, bikes_model_1$fitted.values)
📌 S と RMSE の関係
大きなデータセットでは S と RMSE はほぼ同じ値に近づきます。講義では S＝1509、RMSE＝1507。 どちらも重回帰のほうが小さく、良いと示しました。
🍎 かんたん説明
ダーツで的の中心からどれだけ外れたかを平均したのがRMSE。中心に近いほど（＝Sが小さいほど）上手なプレイヤー＝良いモデルです🎯
F-statistic · モデル全体の検定

「予測変数は全部ゼロ＝無意味」を
否定できるか

summary の最後に出るF統計量（とても小さいp値が科学記法で出る）は、 モデル全体の当てはまりを検定します。ANOVAと同じ考え方でオムニバス検定と呼ばれます。

仮説

H₀：すべての係数がゼロ＝どの予測変数もYと有意な関係がない。
H₁：少なくとも1つの係数はゼロでない。

読み方

Fが大きい＝予測変数を入れたモデルのほうが、入れないモデルよりよく当てはまる。 p値が有意水準（α）より小さければ H₀ を棄却します。

⚠️ 大事な注意
Fが有意でもすべての予測変数が有意とは限りません。これは「モデル全体」を見るオムニバス検定。 「少なくとも1つは有意」とわかるだけで、どれが一番効いているかはさらに分析が必要です。
1summary(bikes_model_1)$fstatistic # F値＋分子/分母の自由度も返る
🍎 かんたん説明
F統計量は「このチーム、誰か1人でも仕事してる？」という質問。答えが「YES」とわかっても、 誰がエースかまではわからない。それは個別に調べます⚽
Mallows' Cp ・ まとめ

MallowのCp と、6指標の早見表

MallowのCpは、計算値を予測変数の数と比べます（数えるときは切片も含む）。 Rに組み込み関数が無いので、データとモデルを渡して自作の関数で計算します。

Cp の値意味
Cp ≒ 予測変数の数✅ 良いモデル。フルモデル並みに説明しつつ、過度に複雑でない（残差もほぼ正規）。
Cp > 予測変数の数⚠️ 過学習（オーバーフィット）
Cp < 予測変数の数⚠️ 過小学習（アンダーフィット）

講義の例：気温モデルは「切片＋気温」でCp＝2、予測変数の数＝2。重回帰モデルも Cp＝予測変数の数。 どちらも「良いモデル」と判定されました。なお、ここで挙げた6つは代表的な指標の一部で、他にも多くの診断があります。

📋 診断指標 まとめ（方向に注目）

指標良いのは何を見る
調整済みR²高い ⬆️無駄な変数を罰した説明力
AIC低い ⬇️当てはまり×複雑さのバランス
BIC（=SBC）低い ⬇️同上（シンプルさをより重視）
S / RMSE低い ⬇️予測の平均的なズレ
F統計量大きい・p小 ⬆️モデル全体が有意か
MallowのCp≒ 予測変数の数過学習 / 過小学習
📌 結論
1つに頼らず複数を総合して比較を。今回はすべての指標で「重回帰モデル」が（少しだけ）良いという結論でした。
🍎 かんたん説明
MallowのCpは「ちょうどいい荷物の量メーター」。多すぎ（過学習）でも少なすぎ（過小学習）でもダメ。 予測変数の数と同じくらいがベストな旅行カバンです🧳
