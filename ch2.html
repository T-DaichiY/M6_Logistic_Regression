📐 ロジスティック回帰モデルの仕組み

係数を理解するために必須の3変換：確率・オッズ・対数オッズの関係を完全マスターする。

① 3つの変換② 確率とオッズ③ 対数オッズ(Logit)④ シグモイド関数📝 Quiz
①

3つの変換を理解する

確率 → オッズ → 対数オッズの流れ

ロジスティック回帰の係数を理解するには、3つの概念を順番に理解する必要があります。

確率 (Probability)
0 〜 1
→
オッズ (Odds)
0 〜 ∞
→
対数オッズ (Log Odds)
−∞ 〜 +∞

これらの変換を経ることで、「確率を予測する」という問題を、線形回帰と同じ形の式で解けるようになります。

🍎 かんたん説明

「確率→オッズ→対数オッズ」は、お金の両替みたいなもの。円をドルに換え、さらにユーロに換える。形は変わるけど、本質は同じ情報を持っています💱

②

確率とオッズ

「50%」と「1対1」は同じ意味

確率は「事象が起こる割合」（0〜1）。オッズは「起こる確率」÷「起こらない確率」です。

確率 Pオッズ P/(1-P)解釈
0.5（50%）0.5/0.5 = 1.0起こる:起こらない = 1:1
0.7（70%）0.7/0.3 ≈ 2.33起こる:起こらない ≈ 2.3:1
0.1（10%）0.1/0.9 ≈ 0.11起こる:起こらない ≈ 1:9
# 確率→オッズの計算（Rで）p <- 0.7odds <- p / (1 - p)odds # 2.333...
📌 オッズ比の特徴：確率が0のとき オッズ=0、確率が0.5のとき オッズ=1、確率が1に近づくにつれてオッズは∞に向かいます。
③

対数オッズ（Log Odds / Logit）

オッズの対数で「−∞〜+∞」に広げる

オッズは0〜∞の範囲ですが、自然対数（log）を取ると−∞〜+∞に広がります。これを対数オッズまたはlogitと呼びます。

# オッズ→対数オッズlog_odds <- log(odds) # log() は自然対数# 確率 0.5 → オッズ 1.0 → 対数オッズ 0log(0.5 / (1 - 0.5)) # → 0# 確率 0.7 → オッズ 2.33 → 対数オッズ 0.847log(0.7 / (1 - 0.7)) # → 0.847

ロジスティック回帰の係数はこの対数オッズのスケールで表されます。これが重要です。

🍎 かんたん説明

確率は0〜1しか動けなくて窮屈。オッズは0〜∞でまだ不便。対数オッズはマイナスもプラスも自由に動けます。回帰式と組み合わせるために、この「自由な動き」が必要なんです🎯

④

シグモイド関数の式

ロジスティック回帰の核心
📌 3変換を一本につなぐ：確率 p → オッズ p/(1−p) → 対数オッズ log(p/(1−p))。この対数オッズ（logit）を予測変数の一次式 β₀+β₁x でモデル化するのがロジスティック回帰です。逆向きにたどればシグモイドで確率に戻せます。「係数は対数オッズの世界の言葉」だと覚えておくと、後の章の係数解釈がすべて一貫して理解できます。

ロジスティック回帰の予測式はこうなります：

P(Y=1) = 1 / (1 + e−(β₀ + β₁X₁ + β₂X₂ + …))

カッコの中（β₀ + β₁X₁ + …）は、重回帰で見慣れた回帰式と全く同じ形です。それをeの指数乗の形に入れることで、出力が必ず0〜1の確率になります。

処理の段階式範囲
① 線形予測子β₀ + β₁X₁ + …−∞ 〜 +∞
② 対数オッズlogit(P) = 上記と同じ−∞ 〜 +∞
③ 確率（最終出力）1 / (1 + e−logit)0 〜 1
📌 まとめ：係数（β）を得るためには 確率→オッズ→対数オッズ の変換が必要。最終的な確率を得るには その逆（シグモイド関数）で戻します。

📖 用語集

オッズ
Odds
確率Pを(1−P)で割った値。「起こる：起こらない」の比率。
対数オッズ
Log Odds / Logit
オッズの自然対数。−∞〜+∞の範囲を取る。ロジスティック回帰の係数のスケール。
シグモイド関数
Sigmoid Function
1/(1+e^−x)。対数オッズを0〜1の確率に変換する。
e（自然対数の底）
Euler's number
≈2.718。自然対数の計算に使われる定数。
