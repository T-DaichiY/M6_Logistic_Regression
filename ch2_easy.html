一言でいうと

確率→オッズ→対数オッズの3変換が核心

🎯 これだけ覚えればOK

確率を直接回帰式に使えないので「オッズ」→「対数オッズ（logit）」と変換する。この3ステップで、0〜1の確率を−∞〜+∞の回帰式のスケールに合わせられる。最後にシグモイド関数で確率に戻す。

ロジスティック回帰の「仕組み」はこの変換の往復です。一度マスターすれば、係数の解釈も出力の理解も全部つながります。

📊
① 確率 P　　（0〜1）
例：P = 0.70（70%がオス）
↓ オッズ = P / (1−P)
⚖️
② オッズ　　（0〜+∞）
例：0.70/0.30 = 2.33　「1より大きい＝成功が多い」
↓ 対数オッズ = log(オッズ)
📐
③ 対数オッズ（logit）　（−∞〜+∞）
例：log(2.33) = 0.847 ← 回帰係数はこのスケール
①

3変換をリアルタイムで確認

スライダーで確率を変えて変換を体感

スライダーで確率Pを変えると、オッズと対数オッズが即座に変わります。P=0.5を境目に、対数オッズが0から正負に分かれる点に注目してください。

📌 P=0.5のとき：オッズ=1、対数オッズ=0。P>0.5なら対数オッズはプラス、P<0.5ならマイナス。これが「係数の正負」の意味です。
②

Rで手計算してみよう

確率からオッズ・対数オッズへ

コードを手打ちアニメーションで見てみましょう。「▶ コードを打つ」ボタンを押してください。

▶ 確率（P=0.7）からオッズを計算

# 確率からオッズへ
p <- 0.70
odds <- p / (1 - p)
odds  # → 2.333

▶ オッズから対数オッズ → さらに逆変換で確率に戻す

# オッズから対数オッズ（logit）へ
log_odds <- log(odds)
log_odds  # → 0.8473
# 逆変換：対数オッズ → 確率
p_back <- 1 / (1 + exp(-log_odds))
p_back  # → 0.70（元に戻った！）

▶ 重要な基準点：P=0.5のとき

# P=0.5のときを確認
p <- 0.5
odds <- p / (1 - p)  # → 1.0
log_odds <- log(odds)  # → 0（重要な基準点）
# 覚え方：P=0.5 ↔ オッズ=1 ↔ 対数オッズ=0
🍎 ポイント

変換は「往復できる」（可逆）。確率→対数オッズ→確率、と変換して戻ると元の値に戻ります。glm()はこの変換を内部で自動的にやっています。

🤔 答えられるかな？（クリックで答えが開く）

Q1. オッズの計算式は？A. オッズ = P / (1-P)。確率0.7なら 0.7/0.3 ≈ 2.33。「1より大きい＝成功の方が多い」。
Q2. 対数オッズ（logit）とは何？A. オッズの自然対数。log(2.33) ≈ 0.847。ロジスティック回帰の係数はこのスケールで出てくる。
Q3. P=0.5のとき対数オッズはいくつ？なぜ重要？A. オッズ=1、log(1)=0。対数オッズ=0がP=0.5（ちょうど50%）の境目。係数の正負の意味がここにある。

💡 まとめ

• 確率P → オッズP/(1-P) → 対数オッズlog(odds)

• P=0.5のとき対数オッズ=0

• 対数オッズは−∞〜+∞の範囲を持つ

• 変換は可逆（往復できる）
