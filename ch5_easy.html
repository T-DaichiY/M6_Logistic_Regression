🍎 第5章 超やさしい版：モデル評価①：逸脱度

逸脱度（Deviance）= −2LL。スライダーで予測変数を追加して逸脱度が下がる様子を体感・Rコードで確認しよう。

一言でいうと逸脱度の変化を体感Rで確認・検定📉 本編はこちら
⚡

一言でいうと

逸脱度 = モデルの「ズレの大きさ」。小さいほど良い。

🎯 これだけ覚えればOK

重回帰のR²の代わりに「逸脱度（Deviance）= −2 × 対数尤度」でモデルを評価する。小さいほど良いモデル。ヌル逸脱度（変数なし）と残差逸脱度（現在のモデル）の差が「予測変数による改善量」。

📉
逸脱度が小さい → モデルの当てはまりが良い
逸脱度が大きい → モデルが外れている
Deviance = −2 × Log Likelihood（−2LL）

「尤度（ゆうど）」とは「データがこのモデルから生まれた確からしさ」です。尤度が高い（データをよく説明できる）ほど対数尤度は0に近く、逸脱度は小さくなります。

①

ヌル逸脱度 vs 残差逸脱度

スライダーで変数を追加して逸脱度の変化を体感

予測変数を追加するほど残差逸脱度は下がります（モデルが改善する）。ただし変数が多すぎると過学習になるため、AICなど別の指標も必要です。

②

Rで逸脱度を確認・検定する

summary()の読み方とpchisq()による検定

▶ summary()で逸脱度を読む

# モデルを実行してdeviance（逸脱度）を確認
model <- glm(Sex ~ Diameter, data=Abalone, family=binomial)
summary(model)
# Null deviance:     3923.7  on 2834 degrees of freedom
# Residual deviance: 3603.5  on 2833 degrees of freedom

▶ 逸脱度を直接取り出して改善量を計算

# deviance を直接取り出す
model$null.deviance  # ヌル逸脱度
model$deviance       # 残差逸脱度
# 差（改善量）を計算
diff_dev <- model$null.deviance - model$deviance
diff_dev  # → 320.2

▶ カイ二乗検定で「改善が有意か」確認

# 改善量がカイ二乗分布で有意か検定
p_val <- pchisq(diff_dev, df=1, lower.tail=FALSE)
p_val  # → 非常に小さい → 有意！
# あるいはaov()でまとめて検定
anova(model0, model, test="Chisq")
📌 pchisq(diff, df=1, lower.tail=FALSE)のp値が0.05未満なら「予測変数を追加した効果が統計的に有意」。anova(model0, model, test="Chisq")でもっと簡単に確認できます。

🤔 答えられるかな？（クリックで答えが開く）

Q1. 逸脱度の計算式は？A. Deviance = −2 × Log Likelihood（−2LL）。小さいほど良いモデル。
Q2. ヌル逸脱度と残差逸脱度の違いは？A. ヌル=切片のみ（変数なし）のモデルの逸脱度。残差=現在のモデルの逸脱度。その差が予測変数による改善量。
Q3. Rで逸脱度の差を検定するコードは？A. pchisq(diff_dev, df=1, lower.tail=FALSE) または anova(model0, model, test="Chisq")。
