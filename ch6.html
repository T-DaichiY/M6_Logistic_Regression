①

AICとは何か

逸脱度に予測変数数のペナルティを加える

M5でも登場したAIC（赤池情報量規準）は、ロジスティック回帰でも使えます。計算式はほぼ同じです：

AIC = −2LL + 2K
（−2LL = 逸脱度、K = モデルのパラメータ数）
  • −2LL（逸脱度）：当てはまりの悪さを示す部分。小さいほど良い。
  • 2K：予測変数を増やすほど大きくなるペナルティ。複雑すぎるモデルを罰する。
📌 AICは小さいほど良い！　当てはまりが良く（−2LLが小さく）、かつシンプルな（Kが小さい）モデルが低いAICを持ちます。
🍎 かんたん説明

AICは「成績とコストのバランス」で先生を評価するようなもの。どんなに授業が上手くても（当てはまりが良くても）、コストがかかりすぎる（変数が多すぎる）なら減点されます📝

②

2つのモデルをAICで比較

Diameterのみ vs Diameter + Rings

アバロンデータで、2つのモデルを比較します。

# モデル1: Type ~ Diametermodel1 <- glm(Type ~ Diameter, data=abalone2, family=binomial)# モデル2: Type ~ Diameter + Rings（わずかに有意な変数を追加）model2 <- glm(Type ~ Diameter + Rings, data=abalone2, family=binomial)# AICを比較AIC(model1)AIC(model2)
AIC(model1) = 3597.8 (Diameterのみ) AIC(model2) = 3596.1 (Diameter + Rings) 差 = 1.7 ← ほとんど変わらない

RingsはわずかにAICを下げますが、差はとても小さい。AICのペナルティ（2K）が、Ringsを追加することで得られる当てはまりの改善をほぼ相殺しています。

③

AICと逸脱度の違い

ペナルティがある vs ない

AICと逸脱度（Deviance）の最大の違いは予測変数数へのペナルティです。

指標計算式予測変数を増やしたとき
逸脱度（Deviance）−2LL必ず下がる（良く見える）
AIC−2LL + 2K改善が小さければ上がることもある
⚠️ 逸脱度だけ見ると騙される：無意味な変数を追加しても逸脱度は減ります。AICはそれにペナルティを課すので、より公平なモデル比較ができます。
🍎 かんたん説明

逸脱度は「何点取ったか」だけ見る評価。AICは「何点取ったか」と「何問解いたか（変数の数）」の両方を見る評価。変数を増やせば点は上がるが、AICはそれをズルとみなします😏

📖 用語集

AIC
Akaike Information Criterion
赤池情報量規準。= −2LL + 2K。小さいほど良いモデル。
ペナルティ項
Penalty Term
AICの2K部分。予測変数が多いほど大きくなり、過複雑化を防ぐ。
K（パラメータ数）
Number of Parameters
切片 + 予測変数の数の合計。
モデル比較
Model Comparison
複数のモデルを評価指標（AIC・逸脱度など）で比較し最良を選ぶ作業。
