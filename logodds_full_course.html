Why it felt impossible

このレクチャーが「3つの罠」

講師本人も「複雑だ、何度も見直していい」と言っていた箇所。難しいのは内容ではなく、暗黙に飛ばされた3つの前提です。

TRAP 01

「単位」が見えない

係数 -1.30 は確率の話でもオッズの話でもなく、ログオッズ（log odds）の話。同じ出来事を3つの言語で言い直しているだけ、と気づかないと一生ハマる。

TRAP 02

「引き算」だと思ってしまう

係数は2つの状態の差。だが講師は「引き算じゃない、割り算だ」と言う。これは“オッズの世界では割り算 = ログの世界では引き算”という同じ操作の言い換え。ここが最大の山。

TRAP 03

「73%減」の出どころ不明

最後に出てくる「オッズが約73%減る」。これは e係数 = 0.273、つまり「元の0.273倍になる」=「27.3%しか残らない」=「72.7%減る」。1行で説明できる。

The stage · German Credit データ

舞台設定：
「外国人労働者」かどうかで、
良い信用になる確率は変わるか？

qmd の Logistic regression with one binary predictor セクション。予測子は ForeignWorker（外国人労働者か）という二値変数ひとつだけ。Rが吐いた式はこれです。

p = 11 + e−(b₀ + b₁·x)  =  11 + e−(2.11 + (−1.30)·x)

b₀ = 2.11 切片（intercept）＝ x=0 のときのログオッズ。 b₁ = −1.30 係数（coefficient）＝ x が1増えたときの「ログオッズの変化」。今日の主役。

⚠️ このデータの“ねじれ”に注意
このコースのコーディングでは x = 0 が「外国人労働者（foreign）」、x = 1 が「自国民（local）」です。直感と逆なので混乱しやすいポイント。「x が0→1に増える＝外国人から自国民に変わる」と読み替えてください。

The KEY · ここが全部の核心

係数は「引き算」じゃない。
オッズを「割る」と出てくる。

🎲 オッズの世界では「割り算」

2つの状態を比べるとき、オッズは比（ratio）で比べる。
2.25 ÷ 8.25 = 0.273
「新しいオッズは元の0.273倍」。

🤖 ログの世界では「引き算」

同じことをログオッズで見ると、ただの引き算。
0.81 − 2.11 = −1.30
係数とピッタリ同じ。

これが「アハ」の正体 💡
「オッズを割る」と「ログオッズを引く」はまったく同じ操作。
数学のルール ln(A) − ln(B) = ln(A / B) がそれを保証している。
講師が「引き算じゃなく割り算」と言ったのは、確率やオッズという“曲がった世界”では差を取れず、比でしか比べられないから。ログを取って“まっすぐな世界”に移すと、初めてただの引き算になる。
だから回帰係数はログオッズで定義されている。足し算・引き算ができる、唯一まっすぐな言語だからです。
The payoff · 「73%減」の正体

最後の「オッズが約73%減る」
を1行で倒す

レクチャーの締めの一言。実はオッズ比 0.273 をそのまま日本語にするだけです。

考え方はシンプル

オッズ比 = e−1.30 = 0.273。
「新しいオッズ = 元の 0.273倍」という意味。

残る割合 = 0.273 = 27.3%
減った割合 = 1 − 0.273 = 0.727 ≈ 73%

符号で方向がわかる 🔭

オッズ比を覚えるより、符号と1との比較で直感を持つのが速い：

・係数が マイナス → eb < 1 → オッズ減る
・係数が プラス → eb > 1 → オッズ増える
・係数が 0 → e0 = 1 → 五分五分・変化なし

講師の「オッズが等しければ x/x = 1」はこのこと。1からどれだけ離れたかが効果の大きさ。

日本語での最終解釈 📌
「外国人労働者から自国民に変わると（x が0→1）、良い信用であるオッズが約73%下がる」。 ＊確率が73%下がるのではなく、オッズ（比）が0.273倍になる、という点に注意。確率では 0.891 → 0.693 の変化です。

Orientation · まず地図を持つ

「同じところ」と「新しいところ」を
最初に切り分ける

連続予測子が難しく感じるのは、二値と何が違うのか曖昧なまま聞くから。先に分けてしまえば、聞くべきは右側だけになります。

✅ 二値と完全に同じところ

  • 係数はやはりログオッズの変化を表す（単位は同じ）
  • 確率 → オッズ → ログオッズの3つの言語も同じ
  • 係数を出す手順も同じ：新オッズ ÷ 旧オッズ → ln()
  • 符号で方向：プラスで増、マイナスで減、0で変化なし
  • R のコードも glm(Class ~ Age, ...) と形は同一

🆕 連続予測子で新しいところ

  • 「1単位増加」がどの2値でもいい（33→34 でも 55→56 でも）
  • そのオッズ比がどこでも一定（×1.0186）──これが“クール”な核心
  • だから効果は足し算ではなく掛け算で積み上がる（複利的）
  • 一方、確率の変化幅 Δp は一定ではない（S字カーブの場所による）
  • x=0（年齢0歳）は非現実だが、計算の“基準点”として便利

The payoff · 効果は“掛け算”で積み上がる

1年で+1.86%。
では10年、30年では？

オッズ比が一定（×1.0186）ということは、年齢の効果は足し算ではなく掛け算で積み上がるという意味。複利と同じ構造です。スライダーで年数を動かして、オッズが何倍になるか見てみましょう。

確率では一定にならない理由 🧭
オッズは“掛け算の世界”なので一定倍率で素直に伸びますが、確率は0〜1の檻に閉じ込められたS字。 端（0.9超など）に近づくと、いくらオッズを掛けても確率はほとんど動けません。「オッズ比は一定／確率の伸びは頭打ち」──この非対称が連続予測子の一番の落とし穴です。
