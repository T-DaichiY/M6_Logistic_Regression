Chapter 5 · Polynomial Regression

直線では測れない
山なりの関係を、
曲線で捉える。

線形回帰の大前提は「予測変数と応答変数の関係が直線」。でも気温は約65度まで増えて、暑すぎると逆に減る——直線じゃなく放物線。 予測変数を2乗した項を加えれば、曲がった関係もモデルに取り込めます。

Limit · 直線の限界

線形回帰の大前提は
「直線の関係」

線形回帰は、予測変数と応答変数の関係が直線（線形）であることを前提にします。でも現実はいつも直線とは限りません。対処は2つ。

① データを変換する

対数変換（log）などで、データを直線パターンに近づける。変換の種類はたくさんある。

② 非線形の関係を使う（この章）

多項式回帰で、曲線そのものをモデルに取り込む。

🍎 かんたん説明
アイスの売上：寒い→売れない、暖かい→よく売れる、でも暑すぎると外に出たくなくて逆に減る。 これは直線じゃなく「山なりの曲線」。直線の定規ではこの山を測れません🍦
Interactive · 曲線フィッター

同じ点でも、
直線か曲線かで差は歴然

講義では geom_smooth で各予測変数（湿度・風速・気温）を個別にレンタル数と散布図にし、形を確かめます。 結果：湿度はやや曲線、風速もわずかに曲線、気温ははっきりした曲線。 気温は約65度まで上がり、暑すぎるとtaper（先細り）——「外が暑すぎると人は借りない」。下のボタンで直線／曲線を切り替えてください。

Method · 2乗項を加える

予測変数を2乗して、
曲線を表現する

曲がりは2次（quadratic／放物線）っぽい。そこで各予測変数を2乗した列を作ってモデルに加えます。これが多項式回帰です （2乗なので、気温²・風速²の値はかなり大きくなります）。

1bikes <- bikes %>% mutate(humidity2 = humidity^2,
2 temperature2 = temperature^2,
3 windspeed2 = windspeed^2)
4bikes_model_3 <- lm(rentals ~ humidity + temperature + windspeed +
5 humidity2 + temperature2 + windspeed2, data = bikes)
モデル予測変数調整済みR²
気温のみ1個0.50
モデル2気温+湿度+風速0.52
モデル3（多項式・6変数）上記＋3つの2乗項増加！（より説明できた）
📌 ただし注意
モデル3では windspeed2（風速の2乗）のp値が大きく、有意ではありません（星なし）。 散布図でも風速は一番曲がりが弱かった（少しバンプがある程度）。だから風速の2乗は不要かもしれません。
🍎 かんたん説明
直線しか描けないクレヨンに「2乗」という魔法をかけると、急にカーブが描けるように。気温の山も湿度のゆるいカーブもバッチリ🖍️
Refine · 後退消去

いらない項を削る

有意でない予測変数を取り除くことを後退消去（backwards elimination）と呼びます（詳しくは第10章）。 ここでは有意でなかった windspeed2 を削ります。

1# windspeed2（風速の2乗）を取り除く
2bikes_model_3b <- lm(rentals ~ humidity + temperature + windspeed +
3 humidity2 + temperature2, data = bikes)
4# → 残りの予測変数はすべて有意（星つき）に！

削った結果、すべての予測変数が有意になり、調整済みR²は約0.6まで（わずかに）上昇。無駄な項を取り除いて、すっきり良いモデルになりました。

🍎 かんたん説明
料理に入れた調味料のうち「これ、味に関係ないな」というもの（windspeed2）を取り除く作業。余計なものを抜くと、かえって味が引き締まります🧂
Caveat · 多項式のVIF問題

気温と気温²は、
当然相関する

多項式回帰には避けられない問題があります。VIF（許容度）を計算すると、 気温と気温²、湿度と湿度²が多重共線性を示し、VIFが非常に大きくなります。

⚠️ なぜ起きる？
気温²は気温から作った変数なので自動的に相関します（気温なしに気温²は計算できない）。これは構造上当たり前のことです。
📌 重要なポイント
多項式回帰でVIFが高くても、モデルがよく機能し、未知のデータ（テストセット）でもうまく一般化するなら、この共線性を必ずしも除く必要はありません。 気温²のような自然な相関は許容されることが多いのです。

気になるなら、湿度や気温の項を削って調整済みR²が改善するか試せます。講義では湿度を削っても調整済みR²はほぼ変わらず（改善せず）。 一方で湿度と風速は、他の変数が捉えていないシグナルをうまく捉えていました（だから残す価値がある）。

🍎 かんたん説明
「身長」と「身長の2乗」が似ているのは当たり前（同じ人の数字だから）。これは“悪い共線性”ではなく“自然な親子関係”。 テストでちゃんと点が取れる（よく機能する）なら、この親子はそのまま残してOKです👨‍👦
