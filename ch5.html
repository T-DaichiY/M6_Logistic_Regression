📉 モデル評価①：逸脱度（Deviance）

①

モデル評価の考え方

何を見れば「良いモデル」かわかるか

重回帰ではR²やAICでモデルを評価しました。ロジスティック回帰でもモデル全体の当てはまりの良さを評価する必要があります。主に使う指標は：

📉
逸脱度（Deviance）
−2LL。小さいほど良い。
📊
AIC
逸脱度+予測変数のペナルティ。
📏
Z値
個別の予測変数の有意性。
❌
R²
ロジスティック回帰には存在しない。
🍎 かんたん説明

重回帰でR²（決定係数）を使ったように、ロジスティック回帰は「逸脱度」でモデルの良し悪しを測ります。考え方は似ていますが、計算方法が違います。

②

対数尤度と逸脱度

−2LL が何を意味するか

対数尤度（Log Likelihood, LL）は、モデルが観測データをどのくらいうまく説明できているかを示します。重回帰の「残差平方和」に対応する概念です。

  • 対数尤度が大きい（0に近い） → モデルの当てはまりが良い
  • 対数尤度が小さい（大きな負の値） → 当てはまりが悪い

逸脱度（Deviance）は対数尤度に−2を掛けたものです：

Deviance = −2 × Log Likelihood（−2LL）

逸脱度は小さいほど良いモデルです。Rのsummary(glm())では：

  • Null deviance：予測変数なし（切片のみ）のモデルの逸脱度
  • Residual deviance：現在のモデルの逸脱度
③

Rでモデルを比較する

attributes()で値を取り出す

逸脱度の値をRで正確に取り出して比較するには attributes() を使います。

# Model 1: Type ~ Diametermodel1 <- glm(Type ~ Diameter, data=abalone2, family=binomial)# モデルに格納されている全属性を確認attributes(model1)# ヌル逸脱度と残差逸脱度を取り出すnull_dev <- model1$null.devianceresid_dev <- model1$deviance# 逸脱度の差（改善量）dev_diff <- null_dev - resid_devdev_diff # → かなり大きな値
null.deviance = 3924.2 (予測変数なし) deviance = 3595.8 (diameter追加後) diff = 328.4 (改善量)
④

カイ二乗検定で有意性を確認

改善量は統計的に有意か
📌 逸脱度の直感：逸脱度（−2LL）は「モデルがデータをどれだけ説明できていないか」の不一致度で、小さいほど良い当てはまりです。ヌル逸脱度（切片のみ）と残差逸脱度（予測変数あり）の差が大きいほど、その予測変数が効いている証拠になります。差が自由度（追加したパラメータ数）に対して十分大きいかをカイ二乗分布で検定します。

逸脱度の差がカイ二乗分布に従うことを利用して、予測変数を追加したことが有意に改善したかを検定できます。

# 自由度 = 追加した予測変数の数df <- model1$df.null - model1$df.residual# カイ二乗検定のp値p_val <- pchisq(dev_diff, df=df, lower.tail=FALSE)p_val # → 非常に小さい値（有意）
p_value = 2.3e-73 ← 非常に有意（Diameterは有用な予測変数）
📌 解釈：p値が非常に小さい → 逸脱度の減少（改善）は偶然ではない → Diameterを追加することでモデルが有意に改善した。
🍎 かんたん説明

切片だけのモデル（何も知らない状態）と、Diameterを使ったモデルを比べています。「Diameterを知ることでオス・メスが予測できる確率が有意に上がった」と言えます🐚

📖 用語集

逸脱度
Deviance (−2LL)
対数尤度を−2倍した値。モデル評価の基本指標。小さいほど良い。
ヌル逸脱度
Null Deviance
予測変数なし（切片のみ）モデルの逸脱度。ベースラインとして使用。
残差逸脱度
Residual Deviance
現在のモデルの逸脱度。ヌル逸脱度との差で改善量を測る。
pchisq()
Chi-square p-value
Rでカイ二乗分布のp値を計算する関数。lower.tail=FALSEに注意。
