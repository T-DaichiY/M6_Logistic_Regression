🍎 第3章 超やさしい版：ロジスティック回帰のカギ：シグモイド関数

シグモイド関数（Sカーブ）をスライダーとRコードで動かして「どんな値でも0〜1に変換できる」を体感しよう。

一言でいうとSカーブを動かすRで確認する🔑 本編はこちら
⚡

一言でいうと

シグモイド関数（Sカーブ）が「何でも確率に変える」

🎯 これだけ覚えればOK

シグモイド関数 P = 1/(1+e^(-x)) は、どんな実数（−∞〜+∞）を入れても必ず0〜1の確率として出力する。これがロジスティック回帰の核心。X=0でP=0.5、Xが大きいほどP→1、小さいほどP→0。

シグモイド関数は「S字カーブ」の形をしています。左端（Xが極端に小さい）では確率がほぼ0、右端（Xが極端に大きい）ではほぼ1、真ん中（X=0）でちょうど0.5です。

シグモイド関数（Sカーブ）の形
①

Sカーブを動かしてみよう

対数オッズ（X値）が変わると確率がどう変わるか

スライダーでXを変えると赤い点がSカーブ上を動きます。X=0でP=0.5、X>0でP>0.5（1側）、X<0でP<0.5（0側）を確認しましょう。

②

Rで確認する

シグモイド関数を自分で書いてみよう

Rにはシグモイド関数が標準で入っていません（plogis()という関数がありますが、自分で書くと仕組みが理解できます）。

▶ シグモイド関数をRで定義して確認

# シグモイド関数を自分で計算してみる
sigmoid <- function(x) {
  1 / (1 + exp(-x))
}
# x=0のとき → P=0.5（ちょうど中間）
sigmoid(0)
# x=2のとき → P≈0.88（高確率）
sigmoid(2)
# x=-3のとき → P≈0.05（低確率）
sigmoid(-3)

▶ P=0.5で「往復変換」が成り立つことを確認

# P=0.5で往復確認（手計算の再現）
p <- 0.5
odds <- p / (1 - p)  # → 1
logit <- log(odds)     # → 0
p_back <- sigmoid(logit)  # → 0.5（元に戻る）
cat("往復確認：P =", p_back)

🤔 答えられるかな？（クリックで答えが開く）

Q1. シグモイド関数の式は？A. P = 1 / (1 + e^(-x))。xに対数オッズを入れると確率Pが出てくる。Rでは plogis(x) でも計算できる。
Q2. X=0のとき確率はいくつ？なぜ？A. e^0=1なので P = 1/(1+1) = 0.5。X=0は「どちらとも言えない」状態（50%）。
Q3. シグモイド関数の逆関数は何？A. logit関数（対数オッズの計算）。P→logit→sigmoid→P と往復できる。

💡 まとめ

• シグモイド = 1/(1+e^(-x))

• X=0のとき P=0.5（50%）

• Xが大きいほどPは1に近づく

• 変換は可逆（往復できる）
