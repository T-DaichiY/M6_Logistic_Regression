Fundamentals
Residual残差
基本概念

実測値と予測値の差。モデルが「説明できなかった部分」の量。

詳細説明

残差とは、実際に観測されたデータ点（observed value）と、回帰直線による予測値（fitted value / predicted value）の差のことです。数式で表すと residual = observed − predicted（実測 − 予測） となります。残差が正（positive）であれば、実測値が予測値より高いことを意味し、データ点は回帰直線の上側にあります。残差が負（negative）であれば、予測値が実測値より高いことを意味し、データ点は回帰直線の下側に位置します。

Errorとほぼ同義に使われることがありますが、厳密には「Error（誤差）」は理論上の真の誤差ε（イプシロン）を指し、「Residual（残差）」はそれをデータから推定したものです。回帰モデルの診断（Q-Qプロット、残差プロット、Cook's Distanceなど）はすべてこの残差を使って行われます。

residual = yᵢ − ŷᵢ　（実測値 − 予測値）
数値例 気温60°Fのとき、モデルが予測した自転車利用数 = 4,542。実際の利用数 = 4,400。
→ 残差 = 4,400 − 4,542 = −142（負の残差）
→ 予測値(4,542)が実測値(4,400)を上回っている状態
⚠️ よくある間違い（テスト頻出） 「負の残差 = 実測値が予測値より低い（予測値 > 実測値）」→ TRUE。
「負の残差 = モデルの計算が間違い」ではありません。残差は常に存在し、それ自体はエラーではありません。
R
# モデルを作成
model <- lm(rentals ~ temperature, data = bikes)

# 方法① : 残差を取得
residuals(model)

# 方法② : $演算子で取得（同じ結果）
model$residuals

# 残差の確認（最初の6件）
head(residuals(model))

# 残差の基本統計
summary(residuals(model))
# → Min / 1Q / Median / Mean / 3Q / Max が表示される
# → Mean がほぼ0であれば正常（OLSの性質）

# 残差プロット（ランダムに散らばっていればOK）
plot(model$fitted.values, model$residuals,
     xlab = "Fitted Values", ylab = "Residuals",
     main = "Residuals vs Fitted")
abline(h = 0, col = "red", lty = 2)
Least Squares Line最小二乗直線 / OLS
基本概念

全残差の二乗和（RSS）を最小化する回帰直線。Rの lm() が自動計算する。

詳細説明

最小二乗法（OLS: Ordinary Least Squares）は、すべての残差の二乗を合計した値（RSS: Residual Sum of Squares = Σ(yᵢ − ŷᵢ)²）が最小になるような切片β₀と傾きβ₁を数学的に求める手法です。「二乗」するのは、正の残差と負の残差が打ち消し合うのを防ぐためです。Rの lm() 関数は、この計算を内部で自動的に行います。

最小二乗直線はデータに対して唯一存在し、どの他の直線よりもRSSが小さくなるという数学的な保証があります。ただし、これは「モデルが正しい」ことを意味しません。仮定（線形性・等分散性・正規性）が満たされているかどうかは別途確認が必要です。

RSS = Σᵢ (yᵢ − ŷᵢ)²　← これを最小化するβ₀, β₁を求める
R
# 最小二乗回帰モデルを作成
model <- lm(rentals ~ temperature, data = bikes)

# パイプ演算子を使った書き方（同じ結果）
model <- bikes |> lm(rentals ~ temperature, data = _)

# モデルの内容を確認
summary(model)
# → Coefficients（β₀, β₁）
# → Residual standard error（RSE）
# → R-squared / Adjusted R-squared
# → F-statistic

# RSSを手動計算で確認
RSS <- sum(residuals(model)^2)
RSS
Signal vs Noiseシグナル vs ノイズ
基本概念

Signal＝モデルが捉えた意味あるパターン、Noise＝説明できないランダム変動。

詳細説明

Signal（シグナル）とは、データの中にある意味のある構造や傾向のことです。例えば「気温が上がるほど自転車利用者が増える」という関係が回帰モデルで捉えられた場合、これがシグナルです。Noise（ノイズ）とは、シグナルでは説明できないランダムな変動のことで、残差（residuals）として現れます。

データのノイズが高い（= 残差の分散が大きい）ほど、回帰直線による予測精度は下がります。高ノイズのデータに対して無理に当てはめようとするとオーバーフィッティング（過学習）が起きます。R²はシグナルの強さを0〜1で示す指標で、1に近いほどシグナルが強く、ノイズが少ない状態です。

具体的イメージ 気温と自転車利用数 → R² = 0.85 → 変動の85%をシグナルで説明できる（ノイズ15%）
気温と株価 → R² = 0.05 → ほぼノイズ。モデルとしてほぼ意味がない。
β₀ — Intercept切片
基本概念

x = 0 のときのyの推定値。回帰直線がY軸と交わる点。

詳細説明

切片（Intercept）β₀は、すべての予測変数がゼロのときの応答変数の期待値です。回帰式 ŷ = β₀ + β₁x において、x=0を代入するとŷ=β₀になります。ただし、現実のデータではx=0が意味をなさないケースも多い（例：身長=0のときの体重は存在しない）ため、切片の値を単独で解釈することは必ずしも適切ではありません。

Rでの係数の取得方法は2種類あり、coef(model)とmodel$coefficientsのどちらも使えます。出力の1番目が切片（Intercept）です。

ŷ = β₀ + β₁x　→　x=0のとき　ŷ = β₀
R
model <- lm(rentals ~ temperature, data = bikes)

# 全係数を表示（切片 + 傾き）
coef(model)
# (Intercept)  temperature
#  -2179.4         109.7    ← 切片 = -2179.4

# $演算子でも同じ結果
model$coefficients

# 切片だけを取り出す
coef(model)[1]         # または
coef(model)["(Intercept)"]

# 解釈：temperature = 0 のとき rentals = -2179.4 と予測
# ※ 温度が0°Fは現実的ではないため、切片単体の解釈は注意が必要
β₁ — Slope傾き
基本概念

xが1単位増えたとき、yがどれだけ変化するかを示す係数。

詳細説明

傾き（Slope）β₁は、予測変数（x）が1単位増加したときの応答変数（y）の予測変化量です。β₁ = 2 であれば「xが1増えるとyが2増える」、β₁ = -3 であれば「xが1増えるとyが3減る」と解釈します。

複数回帰では各変数に対して独立した傾きが存在します（β₁, β₂, β₃...）。それぞれの傾きは「他の変数を固定したときの、その変数1単位変化のyへの影響」を意味します（偏回帰係数）。

数値例 lm(rentals ~ temperature, bikes) で slope = 109.7
→ 気温が1°F上がると、自転車利用数が約110台増えると予測される
R
model <- lm(rentals ~ temperature, data = bikes)

# 傾きだけを取り出す（2番目の係数）
coef(model)[2]          # または
coef(model)["temperature"]
# → 109.7

# 解釈のポイント：
# "As temperature increases by 1 degree,
#  we'd predict rentals to increase by 109.7"

# 複数回帰の場合（偏回帰係数）
model2 <- lm(rentals ~ temperature + humidity, data = bikes)
coef(model2)
# (Intercept)  temperature    humidity
#   -1850.2       95.3         -24.6
# → 気温を固定したとき: 湿度1単位増→利用数24.6減と予測
Response / Outcome Variable応答変数 / 目的変数
基本概念

予測される側の変数（y）。lm(y ~ x) の左辺。

詳細説明

応答変数（Response Variable）は、モデルで予測・説明しようとしている変数です。従属変数（Dependent Variable）、アウトカム変数とも呼ばれます。Rの lm() 関数では、チルダ（~）の左側に書きます。

見分け方 bikes |> lm(rentals ~ temperature, data = _)
→ rentals（利用台数）が応答変数（予測したい変数）
→ temperature（気温）が予測変数（予測に使う変数）
💡 命名のバリエーション Response variable = Outcome variable = Dependent variable = y variable
すべて同じ概念。教科書やドキュメントによって呼び方が異なります。
Predictor / Explanatory Variable予測変数 / 説明変数
基本概念

予測に使う側の変数（x）。lm(y ~ x) の右辺。

詳細説明

予測変数（Predictor Variable）は、応答変数を予測・説明するために使う変数です。独立変数（Independent Variable）、説明変数（Explanatory Variable）とも呼ばれます。Rの lm() 関数でチルダ（~）の右側に書きます。複数の予測変数を使う場合は + で繋ぎます。

予測変数は連続変数（気温、身長など）だけでなく、名義変数（カテゴリ変数）も使用可能です。カテゴリ変数はRが自動的にダミー変数に変換して扱います。

R
# 単回帰：1つの予測変数
model1 <- lm(rentals ~ temperature, data = bikes)

# 重回帰：複数の予測変数（+で追加）
model2 <- lm(rentals ~ temperature + humidity + windspeed, data = bikes)

# カテゴリ変数（名義変数）も使える
# → Rが自動でダミー変数に変換
model3 <- lm(age ~ height + gender, data = humans)
# gender が factor 型なら gender_Male などのダミー変数が自動生成される
Error Term (ε)誤差項
基本概念

モデルで説明できない残りの変動。理論式ではεで表される。

詳細説明

回帰モデルの理論的な式は y = β₀ + β₁x + ε と書かれ、εが「誤差項」です。εは「モデルが説明しきれない部分」を表し、正規分布に従い、平均0、分散σ²を持つと仮定されます（回帰の仮定の一つ）。実際のデータからはεを直接観察できないため、残差（residual）によって推定します。

y = β₀ + β₁x + ε　（理論式）
ŷ = b₀ + b₁x　（推定式、εは含まない）
💡 ErrorとResidualの違い Error（ε）= 理論上の真の誤差（観測不可能）
Residual = εの推定値（データから計算可能）
テストでは「error」と「residual」はほぼ同義として扱われることが多い

R Functions
summary()要約統計関数
R関数

データセットまたはモデルの要約を返す万能関数。SDは含まれない点に注意。

詳細説明

summary()は用途が2つあります。①データセットに適用すると、各数値変数の Min, 1st Quartile（Q1）, Median, Mean, 3rd Quartile（Q3）, Max と欠損値数（NA's）を返します。②回帰モデルに適用すると、係数（Estimates）, 標準誤差（Std. Error）, t値（t value）, p値（Pr(>|t|)）, RSE, R², Adjusted R², F統計量などが表示されます。

⚠️ テスト頻出の誤解 summary()はStandard Deviation（標準偏差）を返さない！
SDが必要な場合は別途 sd(df$column) を使う。
返されるのは: Min / Q1 / Median / Mean / Q3 / Max の6統計量 + NA's
R
# ① データセットへの適用
summary(bikes)
# → 各列の Min, Q1, Median, Mean, Q3, Max が表示
# → 欠損値がある列は NA's: n も表示される

# ② 単一列への適用（同じ統計量）
summary(bikes$temperature)

# ③ 回帰モデルへの適用（最も多用する）
model <- lm(rentals ~ temperature, data = bikes)
summary(model)
# 出力の見方:
# Coefficients:
#               Estimate Std. Error t value Pr(>|t|)
# (Intercept)  -2179.4    200.1    -10.89   <2e-16 ***
# temperature    109.7      3.4     32.26   <2e-16 ***
#
# Residual standard error: 1281 on 729 degrees of freedom
# Multiple R-squared:  0.6875, Adjusted R-squared:  0.687
# F-statistic: 1601 on 1 and 729 DF,  p-value: < 2.2e-16

age_model <- lm(age ~ height, data = humans)

fitted.values当てはめ値（フィット値）
R関数

モデルが学習データに対して返す予測値。新しいデータへの予測には使えない。

詳細説明

model$fitted.values（またはfitted(model)）は、モデルの学習に使ったデータ（訓練データ）に対する予測値を返します。つまり、すでにモデルが「見た」データに対する当てはめ値です。新しいデータや未知のデータに対して予測を行いたい場合は、fitted.valuesではなくpredict(model, newdata = ...)を使う必要があります。

残差（residuals）は observed - fitted.values で計算されます。

R
model <- lm(rentals ~ temperature, data = bikes)

# 学習データへの当てはめ値を取得
model$fitted.values     # または
fitted(model)           # どちらも同じ結果（731件）

# 残差の確認（observed - fitted）
model$residuals
# → residuals = bikes$rentals - model$fitted.values と一致

# 当てはめ値と実測値の比較プロット
plot(bikes$rentals, model$fitted.values,
     xlab = "Actual Rentals", ylab = "Fitted Values",
     main = "Actual vs Fitted")
abline(a = 0, b = 1, col = "blue")  # 完全予測なら対角線上に並ぶ
⚠️ fitted.values と predict() の違い fitted.values → 学習データのみ（731行のデータ全件分）
predict(model, newdata = ...) → 新しいデータに予測適用可能
テストでは「新しいデータセットへの予測 = predict()」が正解
predict()予測関数
R関数

学習済みモデルを使って新しいデータへ予測を行う。信頼区間・予測区間の出力も可能。

詳細説明

predict()はモデルに新しいデータを渡して予測値を得るための関数です。正しい構文は predict(model, newdata = 新しいデータフレーム) です。第1引数がモデル、newdata引数が新しいデータである点に注意してください（引数の順序に注意）。

interval引数を追加することで、信頼区間（confidence）または予測区間（prediction）も同時に得られます。信頼水準はlevel引数で指定します（デフォルトは0.95）。

R
model <- lm(rentals ~ temperature, data = bikes)

# ① 新しいデータへの予測（基本）
new_data <- data.frame(temperature = 60)
predict(model, newdata = new_data)
# → 4542.1（予測利用台数）

# ② 複数の値を一度に予測
new_temps <- data.frame(temperature = c(40, 50, 60, 70, 80))
predict(model, newdata = new_temps)

# ③ 信頼区間付きの予測（平均応答の範囲）
predict(model, newdata = new_data,
        interval = "confidence", level = 0.90)
#        fit     lwr      upr
# 1  4542.1  4450.9   4634.8
# → 90%信頼区間で (4450.9, 4634.8)

# ④ 予測区間付きの予測（個別観測値の範囲）
predict(model, newdata = new_data,
        interval = "prediction", level = 0.90)
# → 信頼区間より必ず広い範囲になる
⚠️ よくある構文ミス ❌ predict(newdata, model3)（引数の順序が逆）
❌ model3$predict(newdata)（Pythonのメソッド構文）
❌ fitted.values(model3, newdata)（fitted.valuesはnewdataを受け取れない）
✅ predict(model3, newdata = newdata)（正解）
lm.beta() / scale()標準化係数の取得
R関数

変数間のスケールを揃え、予測変数の相対的な影響力を比較するために係数を標準化する。

詳細説明

通常の回帰係数は変数のスケール（単位）に依存するため、そのままでは異なる変数の影響力を比較できません（例：気温の係数109.7と湿度の係数-24.6を直接比較できない）。標準化係数は各変数を平均0・標準偏差1に変換してから求めた係数で、スケール差を排除して影響力を比較できます。

方法は主に2つ：①データを先にscale()で標準化してからlm()を実行する、②モデル作成後にlm.beta()（QuantPsycパッケージ）を使う。標準化係数は影響力の比較のみに使い、実際の予測には使わないという点が重要です。

R
# 方法① : scale()でデータを標準化してからlm()
bikes_scaled <- bikes
bikes_scaled$temperature <- scale(bikes$temperature)
bikes_scaled$humidity    <- scale(bikes$humidity)

model_std <- lm(rentals ~ temperature + humidity, data = bikes_scaled)
coef(model_std)
# (Intercept)  temperature    humidity
#   4504.3       856.2        -220.8
# → temperatureの影響力がhumidityの約4倍

# 方法② : QuantPsycパッケージのlm.beta()
install.packages("QuantPsyc")
library(QuantPsyc)

model_raw <- lm(rentals ~ temperature + humidity, data = bikes)
lm.beta(model_raw)
# temperature    humidity
#   0.7823       -0.1897
# → 絶対値が大きいほど影響力が強い（temperatureが最大）

# ⚠️ 重要：標準化係数は比較専用。予測には使わない。
⚠️ 標準化係数の目的を間違えない 標準化係数 → 変数間の影響力比較のために使う
多重共線性の検出 → VIFを使う（標準化係数ではない）
予測（新データへの適用） → 元スケールの係数を使う
step()ステップワイズ変数選択
R関数

AIC（デフォルト）を基準に変数を自動的に追加・削除しながら最適なモデルを探す。

詳細説明

step()はRの基本関数で、ステップワイズ選択により最適な予測変数の組み合わせを見つけます。デフォルトではAIC（赤池情報量基準）を最小化するように変数を選択します。方向（direction）を指定することで、前進選択（forward）・後退削除（backward）・両方向（both）から選べます。

R
# 両方向ステップワイズ（デフォルト: AIC基準）
full_model <- lm(rentals ~ temperature + humidity + windspeed + season, data = bikes)
best_model <- step(full_model, direction = "both")

# 前進選択（何もない状態から変数を追加）
null_model <- lm(rentals ~ 1, data = bikes)   # 切片のみ
forward_model <- step(null_model,
                       scope = list(upper = full_model),
                       direction = "forward")

# 後退削除（全変数から不要なものを削除）
backward_model <- step(full_model, direction = "backward")

# step()の出力の読み方:
# AIC: 現在のモデルのAIC値
# → ステップごとにAICが小さくなるほうの変数追加/削除を選ぶ
# → "none" が最小になったら終了（これ以上改善できない）

# AICの確認
AIC(best_model)
💡 重要なポイント step()のデフォルト基準はAIC（p値でもR²でもない）
選択結果は使用する指標（AIC/BIC/adjusted R²）によって異なる
ステップワイズで選ばれたモデルが必ずしも「最良」とは限らない

モデル評価指標

Model Metrics
RSE — Residual Standard Error残差標準誤差
評価指標

モデルの「平均的な誤差の大きさ」。小さいほどモデルの当てはまりが良い。

詳細説明

RSE（残差標準誤差）は、回帰モデルの予測がどれくらいの誤差を持つかを応答変数と同じ単位で示します。数式は RSE = √(RSS / (n − p − 1))（RSS = 残差二乗和、n = サンプル数、p = 予測変数の数）です。RSEが小さいほど予測精度が高いモデルと言えます。

summary(model)の出力で Residual standard error: 1281 on 729 degrees of freedom のように表示されます。自由度（degrees of freedom）は n − p − 1 です。

R
model <- lm(rentals ~ temperature, data = bikes)

# RSEを取得する方法
sigma(model)        # 最も簡単
# → 1281.2

# summary()の出力からも確認できる
summary(model)$sigma
# → 1281.2

# 手動計算で確認
n <- nrow(bikes)
p <- 1  # 予測変数の数（単回帰なので1）
RSS <- sum(residuals(model)^2)
RSE_manual <- sqrt(RSS / (n - p - 1))
RSE_manual  # → 1281.2

# 解釈: 気温から自転車利用数を予測する際、
# 平均的に約±1281台の誤差が生じる
R² — R-Squared決定係数
評価指標

応答変数の全変動のうち、モデルが説明できる割合（0〜1）。1に近いほど良い。

詳細説明

R²（決定係数）は0から1の値を取り、1に近いほどモデルがデータをよく説明していることを示します。R² = 0.69 であれば「応答変数の変動の69%をモデルで説明できる（残り31%はノイズ）」と解釈します。

注意点：変数を増やせばR²は必ず上がります（意味のない変数でも）。そのため、変数を追加する際のモデル評価にはR²ではなくAdjusted R²を使うべきです。また、学習データのR²がとても高く（例：0.95）、テストデータのR²がとても低い（例：0.40）場合は過学習（Overfitting）のサインです。

R
model <- lm(rentals ~ temperature, data = bikes)

# R²の取得
summary(model)$r.squared
# → 0.6875 （68.75%の変動を説明）

# Adjusted R²の取得
summary(model)$adj.r.squared
# → 0.6870

# 複数モデルのR²比較
model1 <- lm(rentals ~ temperature, data = bikes)
model2 <- lm(rentals ~ temperature + humidity, data = bikes)

summary(model1)$r.squared    # → 0.687
summary(model2)$r.squared    # → 0.721（変数追加でR²は上昇）
summary(model2)$adj.r.squared  # → 0.720（ペナルティ後も改善→変数追加が有効）
⚠️ 過学習（Overfitting）のサイン Train R² = 0.95、Test R² = 0.40
→ 学習データには過剰適合しているが、未知データへの汎化ができていない
→ 変数が多すぎる、またはモデルが複雑すぎる可能性
Adjusted R²自由度調整済み決定係数
評価指標

変数の数に対してペナルティを課したR²。変数選択の比較に適している。

詳細説明

通常のR²は変数を追加するたびに（たとえ無意味な変数でも）必ず増加します。Adjusted R²は変数数に対してペナルティを課すことで、この問題を解決します。有意に貢献する変数を追加した場合のみ増加し、意味のない変数を追加すると減少（または変化なし）する特性があります。

Adj. R² = 1 − (1 − R²) × (n − 1)/(n − p − 1)
💡 使い分けガイド 単一モデルの評価 → R²
複数モデルの比較（変数数が異なる場合）→ Adjusted R²
モデルの複雑さとのトレードオフ評価 → AIC / BIC
AIC — Akaike Information Criterion赤池情報量基準
評価指標

モデルの複雑さ（変数数）と当てはまりの良さのバランスを測る指標。小さいほど良い。step()のデフォルト基準。

詳細説明

AICは「モデルの当てはまりの良さ」と「モデルの複雑さ（変数の数）」の両方を考慮した指標です。当てはまりが同じなら変数が少ないモデルほど低いAICになります（シンプルさを優遇）。AICの絶対値に意味はなく、複数モデルの比較に使います。小さいほうが良いモデルです。

RのデフォルトのAICの計算は AIC = −2 × log尤度 + 2k（k = パラメータ数）です。step()関数はこのAICを最小化するように変数を選択します。

R
model1 <- lm(rentals ~ temperature, data = bikes)
model2 <- lm(rentals ~ temperature + humidity, data = bikes)
model3 <- lm(rentals ~ temperature + humidity + windspeed, data = bikes)

# 各モデルのAICを比較
AIC(model1)   # → 11245.3
AIC(model2)   # → 11100.8  ← 最小 → 最良
AIC(model3)   # → 11102.1

# 複数モデルを一度に比較
AIC(model1, model2, model3)
#         df      AIC
# model1   3  11245.3
# model2   4  11100.8   ← 最良
# model3   5  11102.1

# step()との関係
best <- step(model3, direction = "both")
# → 内部でAICを計算し、最小のAICになるまで変数を追加・削除
BIC — Bayesian Information Criterionベイズ情報量基準
評価指標

AICより変数数へのペナルティが強い。より簡潔なモデルを選ぶ傾向がある。

詳細説明

BICはAICと同様にモデルの当てはまりと複雑さを評価しますが、パラメータ数のペナルティを 2k（AIC）ではなく k × log(n)（n = サンプルサイズ）として計算します。サンプル数が多い（n > 7）場合はBICのペナルティがAICより強くなり、より少ない変数のモデルを選ぶ傾向があります。

AIC = −2×logL + 2k
BIC = −2×logL + k×ln(n)　← ペナルティがより強い
R
model1 <- lm(rentals ~ temperature, data = bikes)
model2 <- lm(rentals ~ temperature + humidity, data = bikes)

# BICの比較
BIC(model1)   # → 11262.4
BIC(model2)   # → 11122.5

# AICとBICを並べて比較
data.frame(
  Model   = c("model1", "model2"),
  AIC_val = c(AIC(model1), AIC(model2)),
  BIC_val = c(BIC(model1), BIC(model2))
)

F-StatisticF統計量
評価指標

モデル全体の有意性を検定する。「少なくとも1つの予測変数が有意」を検定するが、全変数が有意とは意味しない。

詳細説明

F統計量は「このモデルは何も予測しない切片のみのモデルよりも良いか？」を検定します。p値が有意（例：p < 0.05）であれば、少なくとも1つの予測変数が応答変数に対して有意な関係を持つことが示されます。しかし、すべての変数が有意であることを意味するわけではありません。個々の変数の有意性は、各係数のt値・p値で個別に確認する必要があります。

R
model <- lm(rentals ~ temperature + humidity + windspeed, data = bikes)

summary(model)$fstatistic

予測区間・信頼区間

Intervals
Confidence Interval (CI)信頼区間
区間推定

平均的な応答値の推定範囲。モデル自体の不確かさ（パラメータ推定の誤差）を反映する。

詳細説明

信頼区間は「真の平均応答値がこの範囲に入る確率が信頼水準（例：90%）である」という区間です。例えば「気温60°Fにおける平均的な自転車利用台数の90%信頼区間は (4450.9, 4634.8) 台」と解釈します。これはモデルのパラメータ推定にともなう不確かさだけを反映しています。

信頼区間は個々の観測値ではなく、平均応答値に対する推定です。個々の将来の観測値に対する区間は予測区間（PI）を使います。

解釈の例（テスト Q7 Attempt 1） コード: predict(model, newdata = data.frame(temperature=60), interval="confidence", level=0.90)
出力: fit=4542.1, lwr=4450.9, upr=4634.8
解釈: 「気温が60°Fのとき、平均的な自転車利用台数について、90%の信頼区間は (4450.9, 4634.8) 台である」
R
model <- lm(rentals ~ temperature, data = bikes)

# 信頼区間の取得
predict(model,
        newdata   = data.frame(temperature = 60),
        interval  = "confidence",
        level     = 0.90)   # ← 引数名は "level"（"confidence"ではない）
#        fit     lwr      upr
# 1  4542.1  4450.9   4634.8

# 信頼水準を変えると幅が変わる
predict(model, newdata = data.frame(temperature=60),
        interval="confidence", level=0.95)  # → より広い区間
predict(model, newdata = data.frame(temperature=60),
        interval="confidence", level=0.80)  # → より狭い区間
Prediction Interval (PI)予測区間
区間推定

個々の将来観測値の範囲。モデルの不確かさ＋個体差の両方を含むため、常にCIより広い。

詳細説明

予測区間（PI）は「ある特定の条件（例：気温60°F）のとき、新しい1件の観測値がこの範囲に入る確率が信頼水準（90%など）である」という区間です。信頼区間（CI）がモデルの不確かさのみを扱うのに対し、予測区間はモデルの不確かさ + 個々の観測値の固有のばらつき（ランダムエラー）の両方を考慮します。そのため、予測区間は常に信頼区間より広くなります。

CIとPIの違いまとめ CI（信頼区間）: 平均応答の推定範囲 → モデル不確かさのみ
PI（予測区間）: 個別観測値の予測範囲 → モデル不確かさ + 個体ランダム変動
→ PI は常に CI より広い（これはテスト頻出の事実）
R
model <- lm(rentals ~ temperature, data = bikes)
new_data <- data.frame(temperature = 60)

# 信頼区間（CI）
ci <- predict(model, newdata = new_data,
              interval = "confidence", level = 0.90)
# fit=4542.1  lwr=4450.9  upr=4634.8  ← 幅: 183.9台

# 予測区間（PI）
pi <- predict(model, newdata = new_data,
              interval = "prediction", level = 0.90)
# fit=4542.1  lwr=2434.7  upr=6649.5  ← 幅: 4214.8台（CIより大幅に広い）

# 幅の比較
ci_width <- ci[,"upr"] - ci[,"lwr"]  # → 183.9
pi_width <- pi[,"upr"] - pi[,"lwr"]  # → 4214.8
# PI が CI の約23倍の幅 → 個体差のばらつきが支配的
💡 区間を狭くする条件（Attempt 1 Q8） 信頼区間を狭くするには: 信頼水準を下げる ＋ サンプルサイズを増やす
信頼水準を上げると幅は広がる。サンプルが増えると推定精度が上がり幅が狭まる。

モデルの仮定（OLS前提条件）

Assumptions
Linearity（線形性）OLS仮定①
仮定

予測変数と応答変数の関係が線形（直線的）であること。散布図で確認する。

詳細説明

線形回帰モデルは、予測変数(x)と応答変数(y)の関係が直線的であることを前提にしています。非線形な関係（曲線、指数関数的など）がある場合、モデルは現実を正確に捉えられません。対数変換（log transformation）や多項式回帰（polynomial regression）で対処できます。

R
model <- lm(rentals ~ temperature, data = bikes)

# ① 散布図で線形性の確認
plot(bikes$temperature, bikes$rentals,
     xlab = "Temperature", ylab = "Rentals",
     main = "Linearity Check")
abline(model, col = "red")  # 回帰直線を追加

# ② 残差 vs 当てはめ値プロットでも確認
# → ランダムに散らばっていれば線形性OK
# → U字やアーチ状になっていれば線形性違反
plot(model, which = 1)  # Residuals vs Fitted
Nearly Normal Residuals（正規性）OLS仮定②
仮定

残差がほぼ正規分布に従うこと。Q-Qプロットで視覚的に、Shapiro-Wilk検定で統計的に確認。

詳細説明

OLSの仮定の一つとして、残差（ε）が正規分布に従うことが求められます。「Nearly（ほぼ）」という表現が重要で、完全な正規分布でなくても、著しい歪みや外れ値がなければ多くの場合問題ありません。Q-Qプロットで点が対角線に沿って並んでいれば正規性が満たされていると判断します。

⚠️ よくある誤解（テスト頻出） 「1点だけ正規性から外れている → その観測値を削除すべき」→ FALSE
1点だけを除外することは適切ではない。データ変換や外れ値の原因調査が先。
また、Q-Qプロットを見て「正規性に問題なし」と判断するには慎重さが必要。
R
model <- lm(rentals ~ temperature, data = bikes)

# ① Q-Qプロット（視覚的確認）
qqnorm(residuals(model), main = "Q-Q Plot of Residuals")
qqline(residuals(model), col = "red")
# → 点が赤い直線に沿って並んでいれば正規性OK

# Rの組み込み診断プロット（4種類まとめて）
par(mfrow = c(2,2))
plot(model)
# Which = 1: Residuals vs Fitted （線形性 + 等分散性）
# Which = 2: Normal Q-Q         （正規性） ← 特に重要
# Which = 3: Scale-Location     （等分散性）
# Which = 4: Cook's Distance    （影響点）

# ② Shapiro-Wilk検定（統計的確認）
shapiro.test(residuals(model))
# p-value > 0.05 → 正規性違反なし（帰無仮説を棄却しない）
# p-value < 0.05 → 正規性違反の疑い
Constant Variability（等分散性）OLS仮定③ / Homoskedasticity
仮定

残差の分散が、予測変数の値に関わらず一定であること。違反するとHeteroskedasticityとなる。

詳細説明

等分散性（Homoskedasticity）とは、xの値がどこであっても残差の分散σ²が一定であるという仮定です。「Residuals vs Fitted」プロットを見たときに、点が均等に散らばっていれば等分散と判断します。ファネル状（漏斗形）に広がっていれば等分散性違反（Heteroskedasticity）です。

R
model <- lm(rentals ~ temperature, data = bikes)

# ① 残差 vs 当てはめ値プロット
plot(model$fitted.values, residuals(model),
     xlab = "Fitted Values", ylab = "Residuals",
     main = "Residuals vs Fitted")
abline(h = 0, col = "red", lty = 2)
# → ランダムに散らばっていれば等分散性OK
# → ファネル状（右に行くほど広がる）なら違反

# ② Breusch-Pagan検定（統計的確認）
library(lmtest)
bptest(model)
# p > 0.05 → 等分散性OK
# p < 0.05 → Heteroskedasticity の疑い

詳細説明

残差の独立性とは、一つの観測値の残差が他の観測値の残差と相関していないことを意味します。特に時系列データでは、時間的に隣接するデータが互いに影響し合う（自己相関：autocorrelation）ため、この仮定が違反されやすいです。Durbin-Watson検定で診断します。

Durbin-Watson Testダービン・ワトソン検定
診断

残差間の自己相関（系列相関）を検定する統計検定。DW値が2に近いほど自己相関なし。

詳細説明

Durbin-Watson統計量（DW値）は0から4の範囲を取ります。DW ≈ 2：自己相関なし（独立性OK）、DW < 2（0に近い）：正の自己相関（連続した観測が似た傾向）、DW > 2（4に近い）：負の自己相関（交互に正負が逆転する傾向）です。lmtestパッケージが必要です。

# DW ≈ 2         → 自己相関なし ✅

診断

詳細説明

Cook's Distanceは「もしこの観測値を除外したら、回帰係数がどれだけ変化するか」を数値化した指標です。値が大きい観測値ほど、モデルに与える影響が強いことを意味します。一般的な閾値は1.0（D > 1 → 高影響点として要注意）ですが、4/n（nはサンプル数）を使うこともあります。

plot(model, which = 4)

# 上位5件の影響力が強い観測値

VIF — Variance Inflation Factor分散拡大要因
診断

多重共線性の診断指標。VIF > 5 で多重共線性の疑い、VIF > 10 で深刻な問題。

詳細説明

VIF（分散拡大要因）は、「ある予測変数が他の予測変数とどれだけ相関しているか」を測る指標です。VIF = 1 は共線性なし、VIF = 5〜10 が多重共線性の疑い域、VIF > 10 は深刻な多重共線性を示します。VIFが大きい場合、その変数の係数推定が不安定になり、符号が逆転することさえあります。

VIF = 1 / (1 − R²ⱼ)　← R²ⱼ は変数jを他の変数で回帰したときのR²
R
library(car)
model <- lm(rentals ~ temperature + humidity + windspeed, data = bikes)

# VIFの計算
vif(model)
# temperature    humidity   windspeed
#    1.42          1.11       1.33
# → 全てVIF < 5 → 多重共線性の問題なし ✅

# VIFの解釈ガイド:
# VIF = 1       → 完全に独立。共線性なし
# VIF = 1〜5    → 許容範囲内
# VIF = 5〜10   → 多重共線性の疑い ⚠️
# VIF > 10      → 深刻な多重共線性 ❌

# 対処法① : VIFの高い変数を除外
# 対処法② : 共線性のある変数を合成（PCAなど）
# 対処法③ : 正則化手法（Ridge回帰、LASSO）の使用
⚠️ よくある誤解 「相関行列（correlation matrix）だけで多重共線性を検出できる」→ FALSE
相関行列は2変数間の関係しか見られない。3変数以上の複合的な共線性を検出するにはVIFが必要。
Multicollinearity多重共線性
診断

複数の予測変数が互いに強く相関している状態。係数推定が不安定・信頼できなくなる。

詳細説明

多重共線性とは、複数の予測変数が互いに高い相関を持つ状態です。例えば「気温」と「体感温度」を同時に予測変数として使うと、両者がほぼ同じ情報を持つため、それぞれの係数を個別に推定することが困難になります。結果として：係数の標準誤差が大きくなる、係数の符号が意味不明になる、t値が小さくなりp値が大きくなる（本来有意なのに有意でないと判定される）などの問題が起きます。

多重共線性の結果 影響①: 係数が膨張（inflated）または符号が逆転する
影響②: 標準誤差が大きくなり、t値が小さくなる（有意差が出にくい）
影響③: モデルの解釈が困難になる
※ ただし予測精度（R²など）は大きく下がらないこともある

Overfitting過学習
改善

学習データに過剰適合し、未知データへの汎化ができていない状態。Train R²とTest R²の大きな乖離が目印。

詳細説明

過学習は、モデルが学習データの「ノイズ」まで覚えてしまい、新しいデータに対してうまく予測できない状態です。予測変数が多すぎる場合や、モデルが複雑すぎる場合に起こります。典型的なサイン：学習データのR² = 0.95、テストデータのR² = 0.40（大幅な乖離）。

R
set.seed(42)

# データ分割（80% 学習 / 20% テスト）
n     <- nrow(bikes)
idx   <- sample(1:n, size = floor(0.8 * n))
train <- bikes[idx, ]
test  <- bikes[-idx, ]

# 過学習しやすいモデル（変数が多すぎる）
model_complex <- lm(rentals ~ temperature + I(temperature^2) +
                     humidity + windspeed + season +
                     temperature:humidity + temperature:season,
                     data = train)

# 学習データのR²
summary(model_complex)$r.squared         # → 0.84（高い）

# テストデータのR²（実際の汎化性能）
pred_test <- predict(model_complex, newdata = test)
actual    <- test$rentals
SS_tot    <- sum((actual - mean(actual))^2)
SS_res    <- sum((actual - pred_test)^2)
R2_test   <- 1 - SS_res/SS_tot
R2_test   # → 0.62（大幅低下 = 過学習）
Parsimonious Model簡潔なモデル（節約的モデル）
改善

必要最低限の変数で最大限の説明力を持つモデル。複雑さと精度のバランスが鍵。

詳細説明

Parsimony（節約性）は統計モデリングの重要な原則です。オッカムの剃刀（Occam's Razor）の考え方で「同じ説明力なら、より単純なモデルを選ぶ」。変数を増やしすぎると過学習が起き、モデルの解釈が難しくなり、AICやBICも悪化します。AICやAdjusted R²を使った変数選択がParsimonious Modelを目指す実践的な方法です。

Parsimonious Modelの定義（テスト Q19 Attempt 3） ✅ 「簡潔さと効果のバランスを取るモデル」
✅ 「主要なパターンを捉えつつ、必要最低限の変数だけを含むモデル」
❌ 「過学習を減らすためだけに変数を削除するモデル」（説明力を犠牲にしてはいけない）
❌ 「精度を最大化するために全変数を使うモデル」（複雑すぎ）

変数選択法

Variable Selection
Forward Selection前進選択法
変数選択

空のモデル（切片のみ）から始め、モデル改善に最も貢献する変数を1つずつ追加していく方法。

詳細説明

前進選択（Forward Selection）は変数選択の一手法です。最初は切片のみのモデルからスタートし、各ステップで「追加するとAIC（または他の指標）を最も改善する変数」を1つずつ追加します。追加してもモデルが改善しなくなったところで停止します。後退削除（Backward Elimination）と逆のアプローチです。

R
# 前進選択の実装
null_model <- lm(rentals ~ 1, data = bikes)  # 切片のみ
full_model <- lm(rentals ~ temperature + humidity + windspeed + season,
                 data = bikes)

# Forward selection（AIC基準）
forward_model <- step(null_model,
                       scope = list(lower = null_model,
                                    upper = full_model),
                       direction = "forward")

# step()の出力例:
# Step 1: +temperature  AIC = 11245 → 11100 （最大改善）
# Step 2: +humidity     AIC = 11100 → 10950
# Step 3: +season       AIC = 10950 → 10920
# Step 4: +windspeed    AIC = 10920 → 10922 （改善なし → 停止）

# 最終モデルの確認
summary(forward_model)
formula(forward_model)  # 採用された変数を確認
💡 重要なポイント（テスト頻出） 選択結果は使用する指標（AIC / p値 / Adjusted R²）によって異なる
同じ前進選択でもAIC基準とp値基準では異なるモデルになることがある
Stepwise Selection — step()ステップワイズ選択
変数選択

前進選択と後退削除を組み合わせ、AICが最小になるまで変数の追加・削除を繰り返す。

詳細説明

双方向ステップワイズ（Both-direction Stepwise）は最も一般的な変数選択法です。各ステップで「変数を追加する場合」と「変数を削除する場合」の両方を評価し、AICを最も改善する操作を選択します。前進選択だけでは見逃す変数の組み合わせも発見できる点が利点です。

R
full_model <- lm(rentals ~ temperature + humidity + windspeed + season,
                 data = bikes)

# 双方向ステップワイズ（direction = "both"）
best_model <- step(full_model, direction = "both")

# 後退削除のみ

Standardized Coefficients標準化係数
変数選択

スケールを揃えた係数で変数間の影響力を比較するためのもの。予測には使わない。

詳細説明

予測変数のスケール（単位）が異なる場合（例：気温 vs 湿度（0〜1）vs 風速）、元の係数を直接比較しても意味がありません。標準化係数（Beta coefficients）はすべての変数を平均0・標準偏差1にスケールしてから計算した係数で、絶対値が大きいほど影響力が強い変数です。

例（Attempt 1 Q14） β̂₁ = 0.5703、β̂₂ = −0.7294、β̂₃ = 0.1586
→ 絶対値比較: |−0.7294| > |0.5703| > |0.1586|
→ β̂₂（= −0.7294）が最も影響力が強い変数
⚠️ テストでの間違い（Attempt 2 Q14） 「標準化係数 → 多重共線性の検出に使う」→ ❌ 誤り
標準化係数の正しい用途は「変数間の影響力の比較」
多重共線性の検出 → VIF を使う
標準化係数で予測をしてはいけない（元スケールの係数を使う）

データ準備・分割

Data Prep
Train / Test Split訓練データ・テストデータの分割
データ準備

モデルの汎化性能を評価するためにデータを学習用とテスト用に分割する手法。

詳細説明

モデルを学習に使ったデータで評価すると、過学習を見抜けません。そのため、データを学習データ（Training Set）とテストデータ（Test Set）に分割し、テストデータは学習に使わず評価専用にします。一般的な比率は80:20または70:30です。テストデータで評価したR²やRSEが「本当の予測性能」です。

R
set.seed(42)   # 再現性のために乱数シードを設定

n     <- nrow(bikes)           # 総サンプル数 = 731
idx   <- sample(1:n, size = floor(0.8 * n))  # 80%をランダム選択

# データ分割
train <- bikes[idx, ]          # 学習データ（585行）
test  <- bikes[-idx, ]         # テストデータ（146行）

cat("Training samples: ", nrow(train), "\n")  # → 585
cat("Test samples:     ", nrow(test),  "\n")  # → 146

# モデルの学習（学習データのみ使用）
model <- lm(rentals ~ temperature + humidity, data = train)

# テストデータへの予測
pred_test <- predict(model, newdata = test)

# テストデータでのRSE
test_residuals <- test$rentals - pred_test
test_RSE <- sqrt(mean(test_residuals^2))
cat("Test RMSE: ", test_RSE, "\n")

# テストR²の計算
SS_res <- sum((test$rentals - pred_test)^2)
SS_tot <- sum((test$rentals - mean(test$rentals))^2)
R2_test <- 1 - SS_res/SS_tot
cat("Test R²: ", round(R2_test, 3), "\n")
Data Leakageデータ漏洩
データ準備

テストセットの情報が学習プロセスに混入すること。モデルの性能を過大評価させる原因。

詳細説明

データ漏洩は、「本来テスト時には知らないはずの情報」が学習段階に使われてしまう状態です。例えば、分割前に全データでスケーリング（scale()）を行うと、テストデータの統計情報（平均・SD）が学習データの処理に混入します（これは漏洩）。正しくは、スケーリングのパラメータを学習データだけで計算し、テストデータに適用します。

R
set.seed(42)
idx   <- sample(1:nrow(bikes), floor(0.8*nrow(bikes)))
train <- bikes[idx, ]
test  <- bikes[-idx, ]

# ❌ 悪い例（データ漏洩あり）
# 全データでスケーリング → テスト情報が学習に漏洩
bikes_scaled <- scale(bikes[, c("temperature","humidity")])  # 全データで計算
# このスケーリングパラメータ（平均・SD）はtestデータを含む → 漏洩！

# ✅ 正しい例（漏洩なし）
# 学習データのみでパラメータを計算
train_mean <- colMeans(train[, c("temperature","humidity")])
train_sd   <- apply(train[, c("temperature","humidity")], 2, sd)

# 学習データのスケーリング
train_scaled <- scale(train[, c("temperature","humidity")])

# テストデータには学習データのパラメータを適用
test_scaled <- sweep(test[, c("temperature","humidity")],
                     2, train_mean, "-")
test_scaled  <- sweep(test_scaled, 2, train_sd, "/")
Data Cleaning Before Split分割前のデータクリーニング
データ準備

欠損値処理・外れ値除去などのデータ整備は、Train/Testへの分割より前に行うのが原則。

詳細説明

データクリーニング（欠損値の補完・削除、外れ値の処理、型変換など）はデータ分割の前に実施します。クリーニング後のデータを分割することで、学習データとテストデータが同じ品質になり、モデルが最良の性能を発揮できます。ただし、スケーリングや標準化は学習データのみのパラメータで行い（上記Data Leakage参照）、テストデータへはそのパラメータを適用します。

R
library(dplyr)

# 推奨ワークフロー

# ① データクリーニング（分割前）
bikes_clean <- bikes |>
  filter(!is.na(rentals), !is.na(temperature)) |>  # 欠損値除去
  filter(temperature > 0) |>                        # 外れ値除去
  mutate(season = as.factor(season))                # 型変換

# ② データ分割（クリーニング後）
set.seed(42)
idx   <- sample(1:nrow(bikes_clean), floor(0.8*nrow(bikes_clean)))
train <- bikes_clean[idx, ]
test  <- bikes_clean[-idx, ]

# ③ モデル学習（学習データのみ）
model <- lm(rentals ~ temperature + humidity, data = train)

# ④ モデル評価（テストデータ）
pred  <- predict(model, newdata = test)
RMSE  <- sqrt(mean((test$rentals - pred)^2))
cat("Test RMSE:", round(RMSE, 1))
