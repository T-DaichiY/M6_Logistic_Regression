🎯 これだけ覚えればOK

German Credit（1000件の信用審査データ）でロジスティック回帰を実践。Good（良い信用）=1、Bad=0。P(Good)=0.7から対数オッズを計算し、glmの切片（0.847）と一致するを確認。contrasts()で参照カテゴリを必ず確認すること。

🏦 German Credit データの概要

700
✅ Good（=1）
300
❌ Bad（=0）
1,000

①

データの確認とcontrasts()

参照カテゴリを間違えると解釈が逆になる

ロジスティック回帰では「どちらを1（=予測対象）にするか」が重要です。contrasts()で必ず確認しましょう。

▶ データ読み込みと基本確認

# library読み込みとデータ確認
library(caret)
data(GermanCredit)
str(GermanCredit)    # 変数の構造を確認
head(GermanCredit)
table(GermanCredit$Class)  # Good:700, Bad:300

▶ contrasts()で参照カテゴリを確認

# contrasts()で参照カテゴリを確認（重要！）
contrasts(GermanCredit$Class)
#      Good
# Bad     0  ← Bad=0（参照）
# Good    1  ← Good=1（予測対象）
# → P(Good)を予測するモデルになる
⚠️ contrasts()を確認しないと「Goodを予測しているのかBadを予測しているのか」が分からなくなります。係数の解釈が逆になる大きなミスにつながります。

切片の確認とモデル拡張

切片＝対数オッズ → 手計算で再現できる

▶ 切片のみモデルで対数オッズを確認

# 切片のみモデルでP(Good)の対数オッズを確認
m0 <- glm(Class ~ 1, data=GermanCredit, family=binomial)
coef(m0)  # → (Intercept) = 0.8473
# 手計算で確認
p <- 700/1000
odds <- p/(1-p)     # → 2.333
log(odds)         # → 0.8473 ← 切片と一致！

▶ ForeignWorker変数を追加して拡張

# バイナリ予測変数ForeignWorkerを追加
m1 <- glm(Class ~ ForeignWorker, data=GermanCredit, family=binomial)
summary(m1)
# ForeignWorkerYes の係数から
# P(Good | 外国人労働者) を計算できる
p_foreign <- plogis(coef(m1)[1] + coef(m1)[2])
p_foreign  # → ? 確認してみよう
📌 plogis()はシグモイド関数（1/(1+e^-x)）のRの組み込み関数です。対数オッズを確率に変換するときに使えます。

🤔 答えられるかな？（クリックで答えが開く）

Q1. German Creditでは何を予測している？A. 信用が「Good（良い）」である確率。contrasts()でGood=1を確認してから解釈する。
Q2. P(Good)=0.7のとき対数オッズは？A. オッズ=0.7/0.3=2.333、log(2.333)≈0.847。これがglmの切片と一致する。
Q3. なぜcontrasts()を必ず確認するのか？A. どちらのカテゴリが1（予測対象）かで係数の解釈が逆になるから。GoodとBadのどちらを「1」としているかを必ず確認する。
