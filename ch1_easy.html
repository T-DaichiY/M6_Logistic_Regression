一言でいうと

ロジスティック回帰＝「どちらか？」を確率で予測する機械

🎯 これだけ覚えればOK

重回帰は「いくつ？（連続値）」を予測。ロジスティック回帰は「どちら？（0か1か）」を確率で予測する。答えは必ず0〜1の確率として出てくる。たとえば「がんである確率72%」「スパムである確率91%」。

ロジスティック回帰は名前に「回帰」とありますが、実際には分類問題を解くモデルです。「2つのグループのうち、どちらに属するか」を確率として予測します。

①

なぜ重回帰ではダメ？

確率は0〜1の間でなければおかしい

「スパムかどうか（1=スパム、0=正常）」をそのまま重回帰で予測しようとすると、予測値が−0.3や1.7になることがあります。これは確率として意味をなしません。

❌ 重回帰の問題点

予測値 = −0.3 ～ 1.7
0未満・1超えが出てしまう
確率としておかしい！

✅ ロジスティック回帰

予測値 = 0.0 ～ 1.0
必ず確率の範囲に収まる
シグモイド関数のおかげ！

ロジスティック回帰はシグモイド関数（Sカーブ）を使って、どんな数値でも必ず0〜1に押し込みます。これが最大の特徴です。

▶ まず重回帰との違いをコードで見てみよう

# 重回帰（比較用）
lm(bike_rentals ~ temp + humidity + windspeed, data=df)
②

確率を体感してみよう

スライダーを動かして「バイナリ判定」を確認

スライダーで確率を変えると、モデルが「0（否定）」か「1（肯定）」どちらに判定するかがリアルタイムで変わります。デフォルトの閾値はP=0.5（50%）です。

※ P≥0.5なら「1」、P<0.5なら「0」と判定（デフォルト閾値）。閾値は用途に応じて変更可能。

🍎 ひとことで

50%未満→「0（否定）」、50%以上→「1（肯定）」。コインの表裏と同じ感覚。医療診断なら「がんかもしれない=1」を見逃さないよう閾値を30%にするなど、目的に合わせて調整します。

③

glm()の使い方

lm()から1か所変えるだけ

Rでロジスティック回帰を実行するのは、重回帰のlm()をglm()に変えて、family=binomialを追加するだけです。式の書き方はまったく同じです。

🔄
重回帰：lm(結果 ~ 変数1 + 変数2, data=df)
ロジスティック：glm(結果 ~ 変数1 + 変数2, data=df, family=binomial)

▶ スパム判定モデルのコード例

# ロジスティック回帰（family=binomialを追加するだけ！）
glm(spam ~ word_freq + capital_run + exclamation,
    data=df, family=binomial)
# family=binomial が「バイナリ結果を予測する」という合図

▶ モデル実行から結果確認までの全体コード

# モデルを実行して結果を確認
model <- glm(spam ~ word_freq + capital_run,
          data=emails, family=binomial)
summary(model)  # 係数・Z値・p値が表示される
exp(coef(model))  # 係数をオッズ比に変換
📌 summary(model)を実行すると係数・Z値・p値がすべて表示されます。exp(coef(model))で係数をオッズ比（解釈しやすい形）に変換できます。

🤔 答えられるかな？（クリックで答えが開く）

Q1. ロジスティック回帰はどんな結果を予測する？A. 2択（バイナリ）の結果が起こる確率。例：スパムである確率、がんである確率。答えは必ず0〜1の間。
Q2. lm()とglm()の違いは？A. glm()にfamily=binomialを追加するだけ。式の書き方（結果 ~ 変数1 + 変数2）はまったく同じ。
Q3. なぜ重回帰をそのまま使えない？A. 重回帰の予測値は0〜1の範囲を超えることがある（例：−0.3や1.7）。確率としておかしいので、シグモイド関数で変換するglmを使う。

💡 まとめ

• 2択（バイナリ）の結果を確率で予測する

• 確率は必ず0〜1の範囲に収まる

• lm()の代わりにglm(family=binomial)

• 重回帰は確率の範囲を超えてしまう
