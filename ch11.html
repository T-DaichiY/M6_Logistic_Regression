前進（足すだけ）と後退（削るだけ）を同時にやったら？それがステップワイズ。 さらに、専門知識で変数を入れる順番を決める「階層的選択」、そして「シンプルが一番」のオッカムの剃刀まで。AICが下がっていく様子を追います。

Idea · ステップワイズとは

両方の「いいとこ取り」

前進選択は足すだけ、後退選択は削るだけ。「両方を同時にやったら？」と思った人——講師いわく「するどい！」。 それがステップワイズ（mixed／混合）選択。前進のように足し、後退のように削る、各ステップで「足すか削るか」を判断します。

前進（forward）

空モデルから足す一方。

後退（backward）

全部入りから削る一方。

ステップワイズ（both）

足す＋削るを行き来できる。

📌 一度削った変数も、また足せる
ステップワイズの強みは、後で「やっぱり戻したい」変数を再追加できること。各ステップで「＋（足す）」候補と「−（削る）」候補の両方を検討し、AICが一番下がる操作を選びます。
🍎 かんたん説明
前進＝「足すだけ」、後退＝「削るだけ」、ステップワイズ＝「足したり削ったり両方できる」。 味見しながら「塩を足す」も「水で薄める」も自由にできるシェフです👨‍🍳
Interactive · 探索アニメ

全部入り(AIC 9160)から、
足し引きしてAICを下げる

講義では全部入りモデル（AIC 9160）から始めます。全部入りなのでまず削るしかできません （weekday Saturday を削る → AIC低下、real_feel を削る…）。途中で「＋（足し戻す）」候補も現れます。 「次のステップ」を押して、AICが下がる様子を見てください。もう下がらなくなったら停止＝最終モデル。

いまの操作
スタート：全部入りモデル
AIC（低いほど良い）
9160
いまモデルに入っている変数
AICの低下ステップ 0 / 5
1# 全部入りモデルから両方向（足す＋削る）で探索
2model_all <- lm(rentals ~ ., data = bikes_encoded)
3step(model_all, direction = "both")
📌 ＋と−が混在する
後退選択の出力は「−」だけ、前進は「＋」だけ。でもステップワイズの出力には「＋」と「−」の両方が現れます。これが「行ったり来たり」の証拠です。
※ 変数名は講義の例（weekday Saturday・real_feel・windspeed²・Monday 等）に基づくイメージ図（AIC値は傾向を示す例）。
🍎 かんたん説明
魔法の言葉は direction = "both"。"forward" でも "backward" でもなく "both"（両方）と言うだけで、足す・削るを自由に行き来してくれます🔀
Control · p値で出入りを決める

olsrr の ols_step_both_p

AICではなくp値を基準にもできます。olsrr の ols_step_both_p なら、 変数が入る基準（p_enter）と出る基準（p_remove）を別々に設定できます。

1library(olsrr)
2# 入る≤0.2、出る>0.1 で除外
3ols_step_both_p(model_all, p_enter = 0.2, p_remove = 0.1)
パラメータ意味
p_enter = 0.2変数がモデルに入るには、p値が0.2以下である必要がある
p_remove = 0.1p値が0.1より大きい変数は、モデルから除外される

出力には AIC・BIC・R²・調整済みR²・SBIC など複数指標が表示されます。講義の例では、 real_feel を追加 → 後で削除、windspeed² を追加 → 後で windspeed を削除、Monday を追加 → やはり不要と判断して削除…と、 足し引きを繰り返して最終モデルにたどり着きます。

⚠️ モデルは自動保存されない
ols_step_both_p は更新後のモデルを自動保存しません。選ばれた変数で実際に使いたいなら、その変数を自分で lm() に書いて新しいモデルを作る必要があります。 表示されるのは標準化係数ではなく、ふつうの係数です。
🍎 かんたん説明
p_enter と p_remove は「クラブの入場・退場ルール」。「p値0.2以下なら入場OK」「0.1超えたら退場」という2つのドアの基準を別々に決められます🚪
Hierarchical · 理論で順番を決める

統計だけでなく
「理論」で積み上げる

階層的回帰は発想が違います。前進・後退・ステップワイズが変数を独立に・統計的に扱うのに対し、 階層的選択は理論的・概念的な構造を尊重して変数を順番に入れます。鍵はドメイン知識（その分野の専門知識）—— データに詳しい専門家が「この変数を先に入れるべき」と、重要度や理論的関係に基づいて順序を決めます。

例：会社の「仕事のパフォーマンス」を予測する

基本属性を入れる（年齢・学歴）── 土台となる変数
仕事関連を追加（経験・スキル）── 直接関係する変数
交互作用を追加（年齢×経験 など）── 最後に積む

このように意味のある順番で積み上げます。これには、その分野とデータへの深い理解が必要です。

🍎 かんたん説明
前進・後退・ステップワイズは「コンピュータが数字（AIC・p値）だけで自動で選ぶ」。 階層的選択は「その道のプロが経験と理論で『この順番で入れよう』と決める」。家を建てるとき、土台→柱→屋根と順番が決まっているのと同じです🏗️
Goal · パーシモニー（簡潔さ）

4手法まとめ ＆ オッカムの剃刀

手法動き判断基準過学習リスク
前進選択足すのみ統計（AIC等）あり
後退選択削るのみ統計（AIC/p値）あり
ステップワイズ足す＋削る統計（AIC/p値）あり
階層的選択理論で順に追加ドメイン知識・理論低め（理論ベース）

パーシモニー（parsimony）＝簡潔さを目指す

変数選択のゴールはパーシモニアスなモデル＝できるだけシンプルかつ効果的にデータを説明し、シグナルを捉えつつ予測変数の数を最小にしたモデルです。

🪒 オッカムの剃刀（Occam's Razor）
「すべての条件が同じなら、よりシンプルな説明（モデル）が好ましい」。性能（AICなど）がほぼ同じなら、変数が少ないモデルを選ぶ。 集めるデータが少なくて済み、解釈しやすく、過学習も避けやすいからです。

効率

変数が少ない＝集めるデータが少なくて済む。

解釈性

少ない変数のほうが人間に理解しやすい。

汎化

変数が少ないほど未知データへの一般化がうまくいく（第12章）。

🍎 かんたん説明
「オッカムの剃刀」は「同じ結果なら、シンプルなほうがエライ」というルール。 旅行カバンも、同じ荷物なら小さいほうがラク。モデルも、同じ性能なら変数が少ないほうが賢いのです🎒
