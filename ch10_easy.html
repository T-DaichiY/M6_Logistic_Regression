🍎 第10章 超やさしい版：バイナリ予測変数①：切片の解釈

contrasts()を確認してから切片を解釈。FW=0（外国人）のとき切片→確率を手計算で再現しよう。

一言でいうとcontrasts()確認切片から確率再現🔀 本編はこちら
⚡

一言でいうと

予測変数を追加すると切片の「条件」が変わる

🎯 これだけ覚えればOK

バイナリ予測変数X₁を追加すると、切片はもはや「全体の対数オッズ」ではなく「X₁=0のときの対数オッズ」になる。GermanCreditでは切片2.11 → exp → 8.25 → 0.891が「外国人労働者（FW=0）の良い信用確率」と一致する。

第9章では切片のみのモデルで切片が「全体の対数オッズ」だと学びました。でも予測変数を加えると変わります。それが「条件付き切片」の考え方です。

📐
logit(P) = β₀ + β₁×X₁
X₁=0 のとき → logit(P) = β₀ のみ
→ 切片 = X₁=0のときの対数オッズ
①

contrasts()を必ず確認

ForeignWorker: 0=外国人、1=国内（直感と逆！）

このデータではForeignWorker: 0=yes（外国人）、1=no（国内）という直感と逆のコーディング。確認しないと解釈が完全に逆になります。

▶ contrasts()でコーディング確認

# contrasts()で必ずコーディングを確認contrasts(GermanCredit$ForeignWorker)#     yes# no    0  ← no=参照（外国人でない）# yes   1  ← yes=予測対象（外国人）# ← 注意：yesが外国人！（直感と逆）
⚠️ 「ForeignWorker=yes」が外国人です。contrasts()を見ると、「yes=1」なので外国人が1。つまり係数がマイナス＝外国人（1）の方が信用が悪い……ではなく、係数のマイナスはFW=1（外国人）と定義された場合の話。混乱したら実際の確率で確認！
②

切片から確率を再現する

FW=0のときの確率を手計算で確認
🏦 ForeignWorker=0 の確率チェーン
切片（対数オッズ）
2.11
オッズ（exp）
8.25
確率
0.891
実データ確認：FW=0の行のGood比率 = 33/37 = 0.891 ✅

▶ 切片から確率を計算するRコード

# ForeignWorker=0のとき（外国人）の確率→切片再現intercept <- 2.1102odds_fw0  <- exp(intercept)        # → 8.25prob_fw0  <- odds_fw0 / (1 + odds_fw0) # → 0.891# 実際のデータ：33/37 ≈ 0.891 と一致！

🤔 答えられるかな？（クリックで答えが開く）

Q1. 予測変数を追加した後、切片は何を表す？A. 「追加した予測変数=0のときの対数オッズ」。バイナリ変数が0の状態の切片。第9章とは意味が変わることに注意。
Q2. ForeignWorker=0のとき確率を求める計算は？A. intercept=2.11 → exp(2.11)=8.25（オッズ） → 8.25/9.25=0.891（確率）。実データの33/37と一致する。
Q3. なぜcontrasts()の確認が必須か？A. ForeignWorkerのコーディングが直感と逆（0=外国人）だから。確認しないと係数の解釈が完全に逆になる。
