Why · なぜ評価が必要か

本当に知りたいのは
「未知データでの成績」

診断（第2章）でモデルの選び方を学びました。でも本当に知りたいのは—— このモデル、見たことのない新しいデータでもうまく予測できる？ ということ。 モデルを本番（production）で使う／デプロイするなら、明日来る新しいデータにもちゃんと予測してほしい。 「学習に使ったデータ」だけでなく「まったく新しいデータ」で良い成績を出せるか——これが汎化（generalization）です。

📌 汎化（はんか）とは
モデルが未知のデータにもうまく対応できる能力のこと。本番で使うモデルは、この汎化性能が高くなければ意味がありません。
🍎 かんたん説明
「過去問の答えを丸暗記」した人は、過去問では満点でも本番の新しい問題では解けません。 モデルも同じ。「練習データで満点」より「本番の新データで何点取れるか」が大事です📝
Interactive · 過学習エクスプローラ

複雑にしすぎると、
テスト成績だけ落ちていく

過学習（overfitting）とは、モデルがパターン（シグナル）を理解する代わりに、手元のデータを丸暗記してしまう状態。 ノイズをシグナルと勘違いします。スライダーで複雑さを上げてみてください。 訓練はどんどん上がるのに、テストはあるピークを過ぎると下がり始めます。

4
※ 概念を示すイメージ図（講義の例：訓練R²=0.95・テストR²=0.4 を再現）
訓練 R²（練習）
0.92
複雑なほど上がり続ける
テスト R²（本番）
0.80
ピークを過ぎると下がる
判定
ちょうど良い
訓練とテストが近い＝良い

📌 講義の例：訓練 R²＝0.95 なのにテスト R²＝0.4 → 過学習のサイン。逆に訓練とテストがだいたい同じなら、過学習していない良いモデルです。

⚠️ 補足
単純な重回帰では過学習はそこまで大きな問題にならないことが多いです。でも今後学ぶ機械学習モデルはもっと複雑で丸暗記しやすい（過学習しやすい）ので、この考え方がとても重要になります。
🍎 かんたん説明
過学習は「教科書を丸暗記したけど意味は理解していない」状態。同じ問題なら満点でも、ひねられると解けない。 モデルにも「丸暗記タイプ」と「理解タイプ」がいて、私たちは理解タイプを作りたいのです🧠
Tool · 訓練・テスト分割

データを2つに分け、
片方は「本番テスト」用に隠す

過学習を見抜く道具が訓練・テスト分割。データを2つに分けます。

全データ 731行 を 80 : 20 に分割
訓練セット 80%（学習）
テスト 20%
約584行：パターンを学ぶ約147行：本番の代わり

訓練セット training set

モデルにパターンや関係を学ばせるためのデータ。

テストセット test set

取っておいて、モデルが見たことのないデータとして性能を評価する。「本番」の代わり。

このクラスでは80:20分割。他のコースでは70:30、85:15、評価（validation）セットを使う構成なども（データの大きさ次第）。 データが少ないときは分割すべきではありません（学習に使えるデータが減りすぎるため）。なお、評価セットやホールドアウトセットもありますが、この章では入門として訓練・テストの2分割に絞ります。

🍎 かんたん説明
問題集を「練習用」と「模擬試験用」に分けるのと同じ。練習用で勉強して、模擬試験用（取っておいた問題）で実力を測る。 最初から答えを見ちゃダメ＝テストセットは最後まで取っておきます📚
In R · 実装する

seedを固定して、80:20に分ける

分割にはランダム性があるので、再現性のために set.seed() でシードを固定します。 これでチームの誰が実行しても、まったく同じ分割・同じ結果を再現できます。

1# ① 再現性のためにシードを固定
2set.seed(123)
3n <- dim(bikes)[1] # 行数 = 731
4train_indices <- sample(1:n, size = 0.8 * n) # 80%の行番号を選ぶ
5train_set <- bikes[train_indices, ] # 選ばれた行
6test_set <- bikes[-train_indices, ] # それ以外（−でマイナス指定）

結果、訓練は約584行（80%）、テストは約147行（20%）に。ちょうど80:20にはならないこともあります。

1# ① 訓練セットでモデルを作る
2model_train <- lm(rentals ~ temperature, data = train_set)
3# ② テストセット（未知データ）で予測
4predictions <- predict(model_train, newdata = test_set)
5# ③ 実測値と予測値の相関からR²を計算
6r_squared <- cor(test_set$rentals, predictions)^2
📌 結果の読み方
講義では、テストのR²が訓練のR²とだいたい同じでした → 過学習なし（ノイズに惑わされずシグナルを捉えている）。 完璧なモデルなら、実測値と予測値が一直線に並び R²＝1。R²は cor() の二乗、または専用パッケージで計算できます。
🍎 かんたん説明
set.seed は「サイコロの出目を固定する魔法」。これがないと実行のたびに違う分け方に。固定すれば誰がやっても同じ結果＝再現できます🎲
Pitfall · データ漏洩

クリーニングは
分割の「前」か「後」か

とても重要なテーマです。「データのクリーニングは、分割の前にやる？後にやる？」 分割の後にやる処理は、まず訓練セットだけで決め、その同じやり方をテストセットにも適用します。

分割の「前」にOK

  • 列名を直す
  • 重複行を削除する
  • 明らかなデータ入力ミスを直す

分割の「後」にやる（テスト情報を使うもの）

  • 欠損値の補完（imputation）
  • 訓練セットだけで決め、同じ処理をテストにも適用
⚠️ データ漏洩（data leakage）とは
テストセットの情報が訓練セットに漏れてしまうこと。これが起きると性能が実際より良く見えてしまう（過度に楽観的）。
講義の例：欠損値を全データの中央値で埋めると、テストのデータが中央値に影響し、それが訓練→モデルに漏れる。 だからNAの補完は分割の後に、訓練セットだけで決めましょう。

訓練とテストの間にはしっかりした仕切り（パーティション）を保つこと。モデルは「訓練データだけ」で学び、「テストデータ（未知）」で評価する——この境界を守るのが鉄則。 また、どんなクリーニングをしたかを記録・説明できるようにしておくのも大切です。
