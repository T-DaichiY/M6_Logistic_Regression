Chapter 3

🔑 ロジスティック回帰のカギ：変換を理解する

確率→オッズ→対数オッズの変換をグラフと手計算で完全理解。シグモイド関数で元に戻る仕組みも確認。

① 変換の全体像② P=0.5で手計算③ 確率を変えて確認④ logitリンクとは📝 Quiz
①

グラフで直感的に理解する

変換のプロセスをS字カーブで視覚化

ロジスティック回帰のカギは、確率(0〜1)という狭い範囲を対数オッズ(−∞〜+∞)という広い範囲に変換し、線形回帰の式を使えるようにすること。そして最後にシグモイドで戻すことです。

① 確率 P（0 〜 1）
↓ オッズ変換 P/(1-P)
② オッズ（0 〜 ∞）
↓ 自然対数 log(odds)
③ 対数オッズ（−∞ 〜 +∞）
↓ 線形回帰式と組み合わせて係数を推定
④ シグモイド 1/(1+e⁻ˣ) で逆変換
↓
⑤ 予測確率 P̂（0 〜 1）✅
🍎 かんたん説明

例えば「確率 → オッズ」は、勝率50%の試合を「引き分けに等しい（1対1）」と言い換えるようなもの。数学的な変換ですが、伝えている情報は同じです。

②

具体的な数値で手計算

p = 0.5 のとき何が起きるか

確率が P = 0.5 のケースで、全変換を手で追ってみましょう。

# Step 1: 確率 → オッズP <- 0.5odds <- P / (1 - P)odds # 0.5 / 0.5 = 1.0# Step 2: オッズ → 対数オッズlog_odds <- log(odds)log_odds # log(1) = 0# Step 3: 対数オッズ → 確率（シグモイドで逆変換）p_hat <- 1 / (1 + exp(-log_odds))p_hat # → 0.5 ✅（元に戻った！）
odds = 1.0 log_odds = 0 p_hat = 0.5

P=0.5 を変換して最後に戻すと、ちゃんと0.5が返ってきます。変換は可逆（reversible）なので、情報は失われていません。

③

確率が変わるとどうなるか

P=0.7 と P=0.1 で確認

他の確率でも同じ変換をしてみます。

確率 Pオッズ対数オッズシグモイドで戻す
0.1（10%）0.1/0.9 ≈ 0.111log(0.111) ≈ −2.20→ 0.1 ✅
0.5（50%）0.5/0.5 = 1.000log(1.0) = 0.00→ 0.5 ✅
0.7（70%）0.7/0.3 ≈ 2.333log(2.333) ≈ 0.847→ 0.7 ✅
0.9（90%）0.9/0.1 = 9.000log(9.0) ≈ 2.197→ 0.9 ✅
📌 重要なパターン：確率が0.5のとき対数オッズ=0。確率が0.5より大きければ正の値、小さければ負の値になります。
🍎 かんたん説明

確率50%は「フィフティフィフティ」で対数オッズ0。50%より確率が上がると対数オッズがプラスになり、下がるとマイナスになる。真ん中（0）が50%という感覚を覚えておきましょう！

④

logitリンクとは何か

なぜ「リンク関数」と呼ぶか
📌 リンク関数の役割：線形予測子 β₀+β₁x（範囲 −∞〜+∞）と、確率の平均（範囲 0〜1）を「橋渡し（link）」するのがリンク関数です。ロジスティック回帰では logit がその橋であり、二項分布の正準リンク（最も自然な選択）です。リンクを変えれば probit など別モデルになりますが、係数が「オッズ比の対数」として解釈できるのは logit ならではの利点です。

この変換（確率→対数オッズ）をlogitリンク関数と呼びます。glm(family=binomial)と書いたとき、Rは自動的にこのlogitリンクを使います。

線形予測子
β₀ + β₁X
logit⁻¹
確率 P̂
0 〜 1
🔵 まとめ：ロジスティック回帰の仕組み
係数（β）は「対数オッズのスケール」で推定される → exp() で「オッズ比」に戻せる → さらに確率に変換できる。この一連の流れが第4章以降の解釈の基礎になります。

Inverse Sigmoid

可逆変換
Reversible Transformation
