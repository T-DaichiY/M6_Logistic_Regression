Chapter 3 · Standardized Coefficients

単位がバラバラなら、
物差しをそろえてから
比べる。

湿度は0〜1、気温は別の尺度、風速はまた別。係数の大きさをそのまま比べても「どれが一番効くか」はわかりません。 Zスコアで単位をそろえ、影響力を公平に比べる方法を、講義の実際の値で学びます。

Problem · なぜ比べられないか

リンゴとオレンジを
比べてはいけない

前章のモデル（湿度・気温・風速でレンタル数を予測）の係数を、そのまま比べるのは厄介です。 講師いわく「単位（スケール）が違うから」。風速・湿度・気温はそれぞれ別の物差しで測られています。

湿度

0〜1 の割合
小さな範囲

気温

温度の尺度
別の物差し

風速

速さの尺度
また別の物差し

⚠️ よくある勘違い
講義のモデルでは湿度の係数が −3100 で一番大きな推定値。「じゃあ湿度が一番重要？」── そうとは言えません。 湿度が小さな範囲（0〜1）だから係数が大きく見えるだけ。しかも全予測変数のp値はとても小さい（みんな有意）。 では本当はどれが一番効いているのでしょう？
🍎 かんたん説明
身長を「センチ」、体重を「キロ」で測って「数字が大きいから身長のほうが大事」とは言えません。 単位が違うものはそのまま比べられない。だからみんな同じ物差しにそろえてから比べます📏
Interactive · 影響力スイッチ

物差しをそろえると、
順位が入れ替わる

ボタンで生の係数と標準化係数を切り替えてください。生では湿度（−3100）が一番大きく見えますが、 単位をそろえると本当の主役＝気温が浮かび上がります（順位は講義どおり：気温＞湿度＞風速）。

※ 生の係数（湿度 −3100）は講義の実値。標準化係数のバー長は順位を示すイメージ図（気温が最大、の関係を表す）。
🍎 かんたん説明
Zスコアは「テストの偏差値」。点数（単位）が違う英語と数学のテストも、偏差値にすれば「どっちが得意か」を 公平に比べられます。標準化＝みんなを偏差値にそろえる作業です📝
3 methods · 方法1・2

手動Zスコア／scale()

3つの方法があります（推奨は方法3）。まず仕組みを理解するため手動で。Zスコア＝(値 − 平均) ÷ 標準偏差。

Zスコアのイメージ（講義の実例）

講義の例：レンタル985件の日のZスコアは−1.8。「平均より約2標準偏差ぶん下」という意味です。

1# 方法1：手動Zスコア（mutateで新しい列。z_ を頭につけて区別、見やすく並べ替え）
2bikes_for_beta <- bikes %>% mutate(
3 z_rentals = (rentals - mean(rentals)) / sd(rentals),
4 z_temp = (temperature - mean(temperature)) / sd(temperature)) # …湿度・風速も同様
5model_std <- lm(z_rentals ~ z_humidity + z_temp + z_wind, data = bikes_for_beta)

符号を無視して絶対値（大きさ）を見ると、気温が一番大きい＝レンタル数に最も影響。 これは2つの別の角度からも裏づけられます：相関（rentals と最も強く相関するのも気温）と、t統計量（summary の3列目で気温が最大の絶対値）。

1# 方法2：scale() なら全列を一発でZスコア化（結果は方法1と同じ・とても簡潔）
2bikes_for_beta2 <- bikes %>% select(rentals, humidity, temperature, windspeed) %>%
3 as.data.frame() %>% scale()
🍎 かんたん説明
方法1が「全部手書きで偏差値を計算」なら、方法2は「計算機のボタン1つで偏差値表を作る」。同じ答えなら、ラクなほうがいい😊
Recommended · 方法3

lm.beta パッケージ（一番おすすめ）

さらに効率的なのが lm.beta。データを標準化する必要がなく、作ったモデルを“引数として”渡すだけで標準化係数が出ます（前2つの方法と違い、データ側をいじらない）。

1# install.packages("lm.beta") が必要
2library(lm.beta)
3lm.beta(bikes_model_1) # 標準化係数（気温が絶対値で最大）
4summary(lm.beta(bikes_model_1)) # 元の推定値＋"Standardized"列を併記
🔢 コードグレード対策
「最良の予測変数の標準化係数」だけを取り出すには、$coefficients の3行目（気温）・2列目（標準化）を抜き出します：
1summary(lm.beta(bikes_model_1))$coefficients[3, 2]
⚠️ "beta" という言葉に注意（4つの意味）
「beta」は多義語で混乱のもと：① 標準化回帰係数の記号（だから lm.beta）、② 一部の教科書では非標準化係数 β₀,β₁,βk、 ③ 母集団の回帰直線の傾き、④ 統計的検出力の話でも登場。課題では「標準化回帰係数」または「lm.beta」と明示します。
🍎 かんたん説明
方法3は「魔法の翻訳メガネ」。データをいじらなくても、メガネをかける（lm.betaに通す）だけで、 バラバラの単位が一瞬で「同じ物差し」に見える！一番ラクで一番おすすめ🪄
Interpret · 解釈と注意点

「X₂はX₁の2倍重要」と
言える。ただし前提つき

標準化係数で β₁＝0.5、β₂＝1 なら、「X₂はX₁の2倍、Yの予測に重要」と言えます。 ただし前提として、両者がだいたい同じ分布で、標準偏差が大きく違わないことが必要です（これは現実には大きな仮定になりうる）。

👍 メリット

  • 元々スケールが違う予測変数の相対的重要度を直接比較できる
  • 変数の重要度をすばやく把握でき、データ探索に役立つ
  • 同じ物差しなので、非専門家にも説明しやすい

👎 注意点（カウション）

  • 実用的な意味が失われることも：「登録1件あたり$100」のほうが「0.2標準偏差」より直感的
  • 予測がやや複雑（標準化を戻す必要）
  • 同じ分布・似た標準偏差という前提が大きな仮定
  • サンプルのばらつきに影響され、他データへの一般化がしにくい
📌 おすすめ
聞き手に応じて、元の係数と標準化係数の両方を報告しましょう。状況によって両方必要になります。 （さらに知りたい人向け：講義は "Pros and Cons of Standardized Regression Coefficients" などのリンクを紹介しています。）
🍎 かんたん説明
偏差値は「学校内の順位」を比べるのに便利だけど、「実際に何点だったか」も大事ですよね。 標準化係数（偏差値）と元の係数（実際の点数）、両方を見せるのが親切なやり方です🎓
