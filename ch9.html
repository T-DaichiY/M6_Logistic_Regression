Chapter 9 · Forward Selection

20変数なら
組み合わせは100万通り超。
だから1つずつ足す。

全部の組み合わせを試すのは無理。「何もない状態」から、最もAICを下げる変数を1つずつ足していく——それが前進選択。 ステップを進めるたびにAICが下がる様子を見ていきましょう。

What · 変数選択とは

データに埋もれた
「シグナル」を見つける

モデルを作るとき、私たちはデータに隠れたパターン（シグナル）を見つけたい。 変数選択は、モデルにとって最良の予測変数＝データを最もよく説明し、予測に役立つ変数を選ぶプロセスです。 ただし「最良」の定義は、組織がどう「良さ」を定義し、どの指標を使うかによります。

🍎 かんたん説明
探偵が事件を解くとき、手がかりは多ければいいわけじゃない。本当に役立つ手がかりだけを選びますよね。変数選択は「使える手がかりだけを厳選する」探偵の技です🔍
Why not · 全探索は無理

組み合わせ爆発

理屈の上では、すべての組み合わせを試す全部分集合回帰が一番正確。でも計算量が爆発します。

予測変数の数試すべきモデルの数（2ⁿ）
3個8通り（手作業でも可能）
10個2¹⁰ ＝ 1,024通り
20個100万通り超！

20個でも100万通り超。現実的ではありません。そこで、もっと効率的な前進選択を使います。

🍎 かんたん説明
20種類のトッピングから「最高の組み合わせ」を全部試食したら、100万通り…お腹がパンクします🍕 だから「一番おいしいトッピングを1つずつ足していく」賢い方法を使うのです。
Interactive · 前進選択アニメ

空っぽから、
AICを最も下げる変数を足す

「次のステップ」を押すごとに、AICを一番下げる変数を1つ追加します。 AICが下がり続け、もう何を足しても下がらなくなったら停止。それが前進選択の最終モデルです。

いまの操作
スタート：空モデル（~ 1）
AIC（低いほど良い）
10000
いまモデルに入っている変数
（まだ無し＝切片だけ）
AICの低下ステップ 0 / 4
1null_model <- lm(rentals ~ 1, data = bikes) # 切片だけ
2full_model <- lm(rentals ~ ., data = bikes) # 全部入り
3step(null_model, scope = list(lower = null_model, upper = full_model),
4 direction = "forward")

※ 変数名とAIC値は前進選択の動き（＋とAIC低下）を示す教材用の例です（最終AIC 9367で停止）。

🍎 かんたん説明
前進選択は「空っぽのリュックに、一番役立つ道具から順に詰めていく」やり方。もう足しても便利にならなくなったら、そこでストップ🎒
step() · 自動化

AICを最小化しながら自動で追加

step関数はデフォルトでAICを最小化します（R²ではなく）。AICは当てはまりと複雑さのバランスを取り、シンプルなモデルを好みます。

⚠️ AICの読み方
AICは相対的に使います。最初のnullモデルのAIC（例10,000）の絶対値自体に意味はありません。step関数は各変数を試してAICが一番下がるものを順に追加し、AICが増え始めたら停止します。

処理の流れ：null → 一番AICを下げる変数（real_feel）を追加 → 次にdate → 次にhumidity²…と続き、これ以上足してもAICが増える点で停止。それが最終モデルです。手動例ではステップ1（1変数）はR²、ステップ2以降（2変数以上）は調整済みR²で比較します。

🍎 かんたん説明
step関数は「自動で道具を詰めてくれるロボット」。役立つ道具（AICを下げる）を順に詰め、「もう何を足してもリュックが重くなるだけ」になったら自動で手を止めます🤖
BIC · 停止基準を変える

もっとシンプルなモデルが欲しいとき

AICの代わりにBICも使えます。BICは複雑さへのペナルティがより強く、よりシンプルなモデルを選びます。Rでは k = log(行数) を指定します。

1step(null_model, scope = list(lower = null_model, upper = full_model),
2 direction = "forward", k = log(nrow(bikes))) # これでBIC
📌 注意
出力には "AIC" と表示されますが、k=log(n) を指定したので実際はBICです。BICのほうが少ない予測変数を選ぶ傾向があります。
⚠️ 前進選択の弱点
前進選択（と次章の後退選択）は多重共線性を考慮しません。すでに似た変数が入っていても相関の高い変数を追加し、冗長になることがあります。 だからモデルを作った後にVIF（第4章）でチェックし、より簡潔（パーシモニアス）なモデルにしましょう。
🍎 かんたん説明
AICは「ふつうの先生」、BICは「もっと厳しくシンプルを求める先生」。どちらも「似た道具のダブり（共線性）」は見てくれないので、最後は自分でVIFチェックを！👩‍🏫
