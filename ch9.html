①

ドイツ信用データの概要

caretパッケージのGermanCreditデータ

今までアバロンデータを使ってきましたが、新しいデータセットで練習します。caretパッケージのGermanCredit（ドイツ信用）データです。

library(caret)data(GermanCredit)# 必要な列だけ抽出GC2 <- GermanCredit[, c("Class", "Age", "Amount", "ForeignWorker")]head(GC2)

②

目的変数の確認

どちらが1でどちらが0か必ず確認

ロジスティック回帰では「どちらが1（成功）か」を必ず確認する必要があります。

# 件数確認table(GC2$Class)# Bad Good # 300 700# コントラスト確認（どちらが1か）contrasts(GC2$Class)# Good# Bad 0# Good 1
Bad = 0 (300件) → 信用評価が悪い Good = 1 (700件) → 信用評価が良い

1（Good）である確率を予測するモデルを作ります。これは直感的ですね（高いほど良い）。

📌 ForeignWorkerの罠：ForeignWorker変数のコーディングが直感と逆の場合があります（1=外国人ではない、0=外国人 など）。必ずcontrasts()で確認しましょう。
③

予測変数なしモデルで確認

切片から基本確率を再現

まず予測変数なしのモデルを実行し、切片から基本確率を確認します（第4章のおさらい）。

# 切片のみのモデルgc_model0 <- glm(Class ~ 1, data=GC2, family=binomial)coef(gc_model0)# (Intercept): 0.8473# 手計算で確認P_good <- 700 / (700 + 300) # = 0.7odds_good <- 0.7 / 0.3 # = 2.333log(odds_good) # = 0.8473 ← 切片と一致！✅
P(Good) = 0.7 (700/1000) odds(Good) = 2.333 (0.7/0.3) log_odds(Good) = 0.847 ← glm切片 0.8473 と一致
🍎 かんたん説明

700人が「Good」、300人が「Bad」。基本確率70%。これが最初のベースラインです。予測変数を追加することで、より正確な予測ができるようになります🏦

④

次のステップ

予測変数を追加して解釈する

第9章ではドイツ信用データの基礎確認まで行いました。次のステップはバイナリの予測変数（ForeignWorker）を追加することです。

# 次のステップ：ForeignWorkerを追加# （第10章以降で詳しく扱います）gc_model1 <- glm(Class ~ ForeignWorker, data = GC2, family = binomial)summary(gc_model1)
🔵 M6のここまでのまとめ：
① ロジスティック回帰 = バイナリ予測に使う
② 確率→オッズ→対数オッズの3変換が核心
③ 切片 = 予測変数がゼロ（または基準カテゴリ）のときの対数オッズ
④ モデル評価は逸脱度・AIC・Z値を使う（R²は使わない）
⑤ 次回（第10章〜）はバイナリ・連続予測変数の係数解釈へ
