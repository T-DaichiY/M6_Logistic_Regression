🍎 第7章 超やさしい版：RでAICを使う：2つのK

数式のK（パラメータ数）とRのk（ペナルティの強さ）は別物。ボタンで違いを確認してRコードで使い方を理解しよう。

一言でいうとkを変えてみる実際の使い方🔧 本編はこちら

一言でいうと

数式のK（パラメータ数）とRのk（ペナルティ強さ）は別物

🎯 これだけ覚えればOK

AICの数式に出てくるK（パラメータ数）と、RのAIC()関数の引数k（ペナルティの強さ）は同じ文字で全く別の意味。Rのkのデフォルトは2で、通常そのまま使えばOK。混乱の原因はここだけ。

K（大文字）
数式の中のK
= パラメータ数
（切片 + 予測変数の数）

例：切片1＋予測変数2 → K=3
Rが自動でカウントする
k（小文字）
Rの引数のk
= ペナルティの強さ
（デフォルト = 2）

例：AIC(model, k=2) ← 変えなくてOK
k=log(n)にするとBICになる
⚠️ 同じ文字なのに意味が全く違う！これだけ覚えれば混乱しない。
①

kの値を変えるとAICはどうなる？

ボタンで切り替えて違いを確認

k=2が古典的AIC、k=log(n)がBIC（ベイズ情報量規準）に相当します。サンプルサイズが大きいほどBICのペナルティが強くなります。

実際の使い方

課題では何も変えなくてOK
✅
AIC(model) だけで古典的AIC（k=2）
課題で「k=2を使う」と指示されても → 何も変えなくてOK

▶ AIC()の使い方とNYCフライト例

# 古典的AIC（k=2、デフォルト）
AIC(model)           # k=2がデフォルト
AIC(model, k=2)    # 同じ結果

# k=log(n) → BIC（ベイズ情報量規準）に相当
n <- nrow(Abalone)
AIC(model, k=log(n))  # ペナルティが強くなる

▶ 2モデルをAIC()で一括比較

# NYC flightsの例でAICを比較
m1 <- glm(delayed ~ dep_hour + month,
       data=flights, family=binomial)
m2 <- glm(delayed ~ dep_hour + month + carrier,
       data=flights, family=binomial)
# AICで比較（課題ではk=2のまま使う）
AIC(m1, m2)

🤔 答えられるかな？（クリックで答えが開く）

Q1. RのAIC()関数の引数kは何を表す？A. ペナルティの強さ（デフォルト=2）。数式のK（パラメータ数）とは全く別の意味。
Q2. AIC(model)とAIC(model, k=2)の結果は？A. 完全に同じ。k=2がデフォルトなので、何も指定しなくても古典的AICが計算される。
Q3. k=log(n)にするとどうなる？A. BIC（ベイズ情報量規準）に相当する値が計算される。サンプルサイズnが大きいほどペナルティが強くなる。
